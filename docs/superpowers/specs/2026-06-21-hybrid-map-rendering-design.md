# 하이브리드 맵 렌더링 설계 (§3-1 개정)

> 상태: 설계 승인됨(2026-06-21, 길중). §3-1 "painted 배경 1장 + 오브젝트 레이어 제외" 결정을
> **3층 하이브리드(바닥 + 구별되는 top-down 오브젝트 + 유닛/VFX)**로 개정한다.

## 1. 배경 / 문제

§3-1(2026-06-13)은 맵을 **painted 배경 1장이 풍경 전부(성벽·건물 포함)를 그리고, 별도 오브젝트
레이어는 두지 않는다**(이중작업)로 확정했다. 그러나:

- **원작 영걸전은 타일 기반이고, 성벽·성문이 별도 지형 타일**이다 — docs/reference/yeonggeoljeon-data.md
  §8-2(HEXZMAP 완전 해석): 타일 인덱스에 `wall`(0xBD–0xC4 등)·`gate`(0xE0–0xE2) 존재. 즉 원작은
  구조물을 배경에 녹이지 않고 **구별되는 오브젝트로** 그린다. §3-1은 여기서 벗어난 선택이었다.
- **전술 게임의 가독성**: "이 칸이 벽인가/엄폐인가/막혔나"를 한눈에 읽혀야 하는데, 배경에 녹은 벽은
  그 판독이 약하다. 순수 painted는 전술 명료성에서 부족하다(2026-06-21 길중 지적).
- **§3-1의 anti-타일 근거("전환·반복·팔레트 seam")는 *바닥(ground)* 문제**다. 듬성듬성 놓는 *오브젝트*
  (벽·문·나무)는 seam 문제가 없다. §3-1은 "타일 바닥"과 "개별 오브젝트"를 뭉뚱그렸다.
- 직접 계기: 에셋보드 K-4 성곽 오브젝트 시트가 `~50° oblique`로 생성돼 아이소(쿼터뷰)로 나왔다.
  시점 버그(top-down이어야 함)이자, 그 김에 "오브젝트를 둘 것인가"를 재검토하게 됐다.

**핵심 데이터 정합**: packages/data/json/terrains.json은 이미 지형 타입이 전술 오브젝트다 —
`wall`(moveCost 99=불가)·`gate`(통행)·`fort`(guard 0.3)·`forest`/`mountain`(엄폐+이동비용)·`cliff`·
`river`·`village`/`barracks`(회복). 오브젝트 레이어를 **지형 구동**으로 짜면 원작과 동형이 된다.

## 2. 목표 / 비목표

**목표**
- 맵 렌더를 3층(바닥 / 구별되는 오브젝트 / 유닛·VFX)으로 개정.
- 오브젝트는 **엄격 top-down**(아이소 폐기), **격자 정렬**, **지형 구동**(+선택 데코 배치).
- 성문/성벽 **파괴 상태**(닫힘→열림→파괴, 온전→붕괴) 표현 — §5 관문돌파·화공이 화면에서 변함.
- 에셋보드 오브젝트 키트(K-4~K-7)를 **복원 + top-down 재작성**.
- painted 배경 역할 축소(바닥·앰비언트만) → 그리드 정합 부담 제거.

**비목표(이번 범위 밖)**
- 깊이정렬(유닛은 항상 맨 위 유지 — §3-1 계승).
- 자유 배치 오브젝트(칸 정렬만). 멀티셀 대형 구조물(성문루 2×2 등)은 후속.
- 엔진 전투 규칙 변경(terrains.json 이동/엄폐 그대로). 본 설계는 **렌더 + 에셋 + 데이터 배치**만.
- back/측면 오브젝트 뷰(top-down 단일 뷰).

## 3. 레이어 모델 (현 렌더러 정합)

현 `BattleRenderer.mount`의 `world` 컨테이너(sortableChildren)는 zIndex로 정렬된다:

| zIndex | 레이어 | 비고 |
|---|---|---|
| −1 | `mapBg` (painted 배경 Sprite) | 있으면 `terrain.visible=false`로 타일 대체 |
| 0 | `TerrainLayer` (바닥 타일) | painted 없을 때의 바닥 |
| 0.5 | gridOverlay | 정합 확인용(기본 off) |
| 1 | `HighlightLayer` | 이동/공격 범위 |
| 1.5 | `ThreatLayer` | 위협범위 |
| **1.8** | **`ObjectLayer` (NEW)** | **구조물·데코 — 바닥 위, 유닛 아래** |
| 2 | `UnitLayer` | 유닛(항상 오브젝트 위) |
| 3 | `FxLayer.world` | 불·연기 등 VFX |

→ **바닥(−1/0) ⟶ 하이라이트/위협(1/1.5) ⟶ 오브젝트(1.8) ⟶ 유닛(2) ⟶ VFX(3).** 오브젝트는
하이라이트보다 위(벽이 범위 틴트를 덮는 게 자연스러움; 벽 칸엔 어차피 이동범위가 안 깔림), 유닛보다
아래(유닛이 벽 앞/위에 섬 — §3-1 "유닛 항상 맨 위" 계승, 격자 정렬이라 오브젝트 간 깊이정렬 불요).

**ObjectLayer는 painted 배경과 무관하게 항상 표시**된다(mapBg는 terrain 타일만 끈다). 이는 **현재
결함을 고친다**: 지금은 painted 배경이 깔리면 `terrain.visible=false`로 TerrainLayer의 지형 데코
(나무·바위·움막 — `buildDecos`)까지 함께 사라져 painted 그림에만 의존한다. 하이브리드에선 데코·구조물이
ObjectLayer로 이관돼 **어느 바닥(painted/타일) 위에서도 유지**된다(§8 마이그레이션).

## 4. 시점 규약 — 엄격 top-down

- **정면 고각 top-down**(원작 영걸전 타일과 동일): **45° 아이소 회전 없음**, 측면 거의 안 보임,
  약한 높이감만(벽 상단 + 살짝의 두께). 바닥·오브젝트·유닛이 같은 평면 규약을 공유.
- 금지: `oblique`, `camera ~N° above`, 두 측면이 보이는 아이소, 긴 대각 블록.
- 에셋보드 오브젝트 프롬프트의 시점 문구를 이 규약으로 통일(§6).

## 5. 오브젝트 레이어 — 데이터 모델

오브젝트는 **두 소스**에서 온다:

### 5.1 지형 구동 피처(주력)
맵 격자의 각 칸 지형 타입에 **피처 스프라이트**가 매핑되면 ObjectLayer가 그 칸에 그린다. 순수
바닥 지형(plain/grass/waste/road 등)은 오브젝트 없음(바닥만).

> **기존 자산 활용**: 이미 `textures.ts` `DECO_FILES`(+`getDeco`/`loadDecos`)와 `TerrainLayer.buildDecos`가
> 일부 지형(나무·바위·움막 등)을 직립 데코 스프라이트로 그린다. ObjectLayer는 이걸 **갈아엎는 게 아니라
> 흡수·격상**한다(§8) — 같은 텍스처 풀을 재사용하고, 거기에 벽/문 오토타일·파괴상태를 더한다.

| 지형 | 오브젝트 | 비고 |
|---|---|---|
| `wall` | 성벽 세그먼트 | **오토타일**: 직교 이웃 wall 연결로 직선/모서리/T/끝 + 외곽 성가퀴 캡 선택 |
| `gate` | 성문 | **상태**: 닫힘/열림/파괴 (§5.3) |
| `fort` | 망루/요새 액센트 | 엄폐 칸 |
| `forest` | 나무/캐노피 | 기존 E-3 캐노피와 통합 가능 |
| `mountain`/`cliff` | 바위/산봉우리/절벽 | 기존 E-2/B-6 통합 |
| `river` | (바닥은 물 텍스처) + 수상전 시 배/선착장 | K-7 |
| `village`/`barracks`/`depot` | 촌락/병영/창고 움막 | 기존 B-8 통합 |

- **오토타일(벽)**: 벽 칸의 스프라이트는 4-이웃 wall 여부 비트마스크로 결정(직선H/V·모서리×4·T×4·
  십자·끝×4·단독). K-4가 직선/모서리/끝 세그먼트를 제공하면 임의 벽 배치를 자동 조립(원작 성벽 캡
  타일과 동형). 매핑표는 구현 플랜에서 확정.

### 5.2 데코 배치(선택)
지형에 안 묶인 순수 시각 요소(군기·잔해·추가 나무·모래주머니)는 맵/스테이지 데이터의 선택 필드
`decorations: [{ cell:[x,y], kind, variant?, flip? }]`로 배치. 비충돌(또는 지형 엄폐 따름). K-5/K-6 키트.

### 5.3 파괴/동적 상태
- **성문**: `closed`(기본) → `open` → `destroyed`. **성벽**: `intact` → `breached`.
- 상태 소스: 스테이지 데이터 초기값 + 런타임 이벤트(§5 관문돌파·화공). 엔진은 이미 통행을 terrain으로
  판정(gate=1·wall=99)하므로, "열림/붕괴"는 **해당 칸 통행 가능화 + ObjectLayer 스프라이트 스왑**으로
  표현(+ K-8 먼지/연기 VFX 1컷). 엔진의 terrain 변경 메커니즘과 결선은 구현 플랜에서(이벤트→칸 상태).
- v1 최소: 스테이지 초기 상태 렌더 + (가능하면) 이벤트 1종(관문돌파) 스왑. 전체 동적화는 후속.

## 6. 오브젝트 키트 — K-4~K-7 복원 + top-down 재작성

2026-06-21에 폐기한 K-4(성곽)·K-5(진영)·K-6(자연)·K-7(수상) 시트를 **git에서 복원**하되:
- 프롬프트 시점 문구를 §4 규약(엄격 top-down, 아이소 금지)으로 재작성.
- 벽/문은 **오토타일 세그먼트 단위**(직선·모서리·끝·성가퀴·문 상태별)로 칸 footprint에 맞게.
- 섹션 K **설명(desc)**의 "폐기" 문구 + B-6/B-7/B-8 검토 노트를 하이브리드로 재서술("top-down 오브젝트
  키트"). 제목의 "+ 오브젝트"도 복원(현재 제목엔 "폐기" 문구 없음 — desc에만 있음).
- 한 칸 = 한 오브젝트 footprint 원칙(멀티셀은 후속).

## 7. painted 배경 역할 축소

- 배경은 이제 **ground + ambient만**(지형 텍스처·원경 산수). **벽·건물·구조물은 안 그림** —
  그건 ObjectLayer가 그린다. 이중작업·이중묘사 방지.
- §3-1의 "그리드가 삽화를 결정"(벽을 칸에 정확히 맞춰 칠하는) 정합 부담 **소멸** — 벽이 격자
  오브젝트라 자동 정렬. blockout→img2img는 바닥 톤·앰비언트용으로 단순화.
- 기존 painted 배경에 벽이 그려져 있으면 이중묘사 — 현재 대부분 placeholder라 마이그레이션 부담 낮음.
  배경 재생성 시 "구조물 빼고 바닥만" 프롬프트로.

## 8. 렌더러 변경 (apps/web/src/pixi)

- **`layers/ObjectLayer.ts` 신설**: `ctx.map` 지형 격자 + `decorations`를 읽어 칸별 오브젝트 스프라이트
  배치(zIndex 1.8). TerrainLayer의 청크/cull 패턴 재사용(대형맵). 오토타일 비트마스크 산출.
- **기존 데코 경로 흡수(마이그레이션)**: `TerrainLayer.buildDecos`(+`chunk.decos`)를 ObjectLayer로 이관하고
  TerrainLayer는 **순수 바닥 타일**로 되돌린다. 이유 ① 이중 묘사 방지(두 레이어가 같은 칸 데코를 그림)
  ② painted 배경 시 `terrain.visible=false`에 데코가 딸려 사라지던 결함 제거(ObjectLayer 항상 표시).
  `textures.ts`의 `DECO_FILES`/`getDeco`/`loadDecos`는 그대로 ObjectLayer가 재사용.
- **`BattleRenderer.mount`**: world에 ObjectLayer 추가(highlights/threat와 units 사이). 텍스처 비동기
  로드(`textures.loadObjects()` 신설) 후 rebake. painted 배경이 있어도 ObjectLayer는 **항상 표시**
  (배경=바닥, 오브젝트=구조물 — 상호 배타 아님; mapBg가 terrain만 끄게 유지).
- **상태 스왑 API**: ObjectLayer에 `setObjectState(cell, state)`(성문/성벽) — 이벤트 결선용.
- 깊이정렬 없음(유닛 항상 위). 입력/충돌은 기존 terrain 기반 그대로(렌더 전용 추가).

- 에셋: `apps/web/public/assets/objects/*`(신규 오브젝트 스프라이트 경로 규약, 컷 도구).

## 10. 검증

- **렌더 회귀**: 기존 27스테이지가 ObjectLayer 추가 후에도 정상 렌더(유닛/이동/하이라이트 무변).
  타일 폴백/ painted 배경 둘 다에서 오브젝트가 올바른 칸에 표시.
- **오토타일**: 합성 벽 배치(직선·ㄱ자·십자·끝)가 올바른 세그먼트로 조립되는 단위 테스트(비트마스크
  → 스프라이트 키).
- **시점**: 생성 오브젝트 시트 시각검수(길중) — 아이소 아닌 top-down, 바닥·유닛과 평면 일치.
- **상태 스왑**: 성문 closed→open 스왑 + 통행 가능화가 한 흐름으로 동작.
- **성능**: 대형맵(적벽급)에서 ObjectLayer cull로 프레임 무저하.

## 11. 열린 항목 / 리스크

- **오토타일 매핑표**(지형 비트마스크→스프라이트 키) 구체화 — 구현 플랜에서.
- **파괴 상태 ↔ 엔진**: 통행 변경을 엔진 terrain에서 어떻게 트리거(이벤트 스키마)할지 — 플랜에서.
- **멀티셀 구조물**(성문루 2×2): 이번 칸-정렬 범위 밖, 후속.
- **기존 painted 배경**에 벽이 박혀 있던 것 재생성 필요 시점.
- **수상전(K-7)**: 배가 "지형 피처"인지 "유닛이 서는 발판"인지 — 후속 수상전 설계와 함께.

## 12. CLAUDE.md §3-1 개정안

§3-1의 2층 확정 문단을 **3층 하이브리드**로 교체:
> "**3레이어로 확정(2026-06-21 §3-1 개정)**: ① 바닥(painted/저대비 시임리스 — *지형·앰비언트만*,
