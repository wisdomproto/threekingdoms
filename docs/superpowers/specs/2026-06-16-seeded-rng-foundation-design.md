# Phase A: 시드 고정 RNG 기반 — 설계

> 날짜: 2026-06-16 · 로드맵 Phase A (A→F, docs/reference/yeonggeoljeon-rifine-combat.md 이식의 토대)
> 결정(길중, 2026-06-16): 전투 모델을 **결정론 → 시드 고정 확률**로 전환. 동기 = 영걸전 리파인
> 전투 깊이(명중/회피·상태이상·회심) 이식. 시드 고정이라 리플레이/리더보드/밸런스 sim 보존.

## 0. 발견 — 레일 대부분 기존재

- `nextRandom(state) → [value, nextState]` 순수 함수형 PRNG: **있음** (`packages/engine/src/rng.ts`).
- `BattleState.rngState`: **있음** (주석 = "미래 RNG 용도로 보존").
- `createBattle(ctx, seed)` 시드 수령 + `RunOpts.seed → createBattle`: **있음** (runner.ts).

→ Phase A = ① 정책 전환 문서화(CLAUDE.md) ② RNG 소비 **계약** 확립 ③ 재현성 테스트. **엔진 동작 불변.**

## 1. 목표 / 비목표

- **목표**: 시드확률 모델로 정책 전환 + *결정적·재현 가능·이벤트 정합*한 RNG 소비 계약 확립.
  Phase A 후 게임 동작 **완전 불변**(소비처 0), 전 패키지 기존 테스트 green.
- **비목표**: 실제 RNG 기능(명중/회피·데미지 분산·상태이상)은 **Phase B+**. 본 Phase는 토대만.

## 2. 설계

### 2-1. RNG 소비 계약 (핵심 — EventPlayer 정합)
- 소비 헬퍼는 Phase B에서 필요한 형태로 추가(`chance(state,p)→{hit,state}`, `range(state,lo,hi)→{n,state}`).
  Phase A는 **계약과 테스트만** 못박는다(소비 0).
- **계약**: RNG로 갈린 결과(명중/빗맞음·분산 피해·상태이상 발동)는 **반드시 BattleEvent에 실어** 프레젠터
  투영이 커밋 상태와 일치하게 한다(dev `diffSnapshot` 단언 유지). 즉 "이벤트가 상태 변화를 전부 서술한다"
  계약을 RNG로 확장 — rngState 전진도 그 이벤트 적용으로 재현된다. (Phase A는 소비 0이라 위반 없음;
  Phase B 첫 소비부터 강제. 위반 시 기존 드레인 단언이 자동 적발.)

### 2-2. 재현성
- `(stage, seed, 행동열)` → 동일 BattleState 진행. `rngState`는 직렬화 가능 정수라 **중도 스냅샷/복원이
  동치**(세이브·리플레이 토대). 세이브스컴 방지 = 같은 상태 같은 롤(리파인 2.3.3 "세이브로드 신공 불가").

### 2-3. 밸런스 sim (§11) — 준비만, 미가동
- `RunOpts.seed`는 이미 흐른다. reportCard의 *시드 분포 → 승률 복원*은 **변량이 생기는 Phase B에서** 추가.
  Phase A는 변량 0이라 어떤 시드든 동일 결과(분포 퇴화 = 현행과 동일). 문서에 "분포 차원 준비됨, 미가동" 명시.

### 2-4. 리플레이 (§14, v1.5) — 토대만
- 리플레이 모듈은 아직 없음(v1.5). Phase A는 "시드+행동열로 재현 가능"만 보장 → 미래 리플레이가 턴로그
  헤더에 시드만 기록하면 성립.

### 2-5. CLAUDE.md SSOT 개정 (docs-first — 이 Phase의 첫 작업)
- **§15 금지목록**: "전투 RNG(치명타·명중%·분산) 영구금지" → **개정**: "**순수(언시드) 랜덤·세이브스컴
  금지. 전투 RNG는 시드 고정만 허용**".
- **§2-1**: 결정론 전제 문단 → "시드 고정 확률(세이브스컴 불가) + 확정 이벤트 혼합". "전투 RNG 금지" 표현 수정.
- **§2-5**: "전투력은 실력으로" → "**기댓값 실력으로**"(한 판은 운, 실력=분포 상단; 시드 고정이라 리더보드/
  리플레이 유효).
- **§11**: "결정론이라 분포 안 나옴 → 매트릭스" 단서 → "**시드 분포로 승률 복원**(매트릭스에 시드 차원)".
- **§7/§9**: 결정론 단언 완화 + docs/reference/yeonggeoljeon-rifine-combat.md & 이식 로드맵(A~F) 포인터.
- **§13 불가침선**: "전투력 영향 랜덤 옵션 금지"는 **현금 확률상품·장비 랜덤스탯**에 한정임을 명확화 —
  시드확률 *전투*는 별개(BM 가드와 무관, 밸런스 sim·리더보드 유지로 정당).
- 이식 로드맵 Phase A~F + 본 스펙 포인터를 §7 하위에 추가.

## 4. 테스트 (핀 vitest)

- **rng**: `nextRandom` 결정성(같은 시드 같은 수열) · state 라운드트립 · 대략 균등.
- **createBattle**: 같은 `(stage, seed)` → 동일 초기 state; `rngState === seed`.
- **재현성**: 같은 `(stage, seed, 행동열)` → 동일 최종 state(스냅샷 동치). 현재는 다른 시드라도 동일
  (소비 0 증명 — Phase B에서 이 테스트가 "시드별 분기"로 갱신될 자리).
- **회귀**: 전 패키지 기존 테스트 green(행동 불변 증명).
