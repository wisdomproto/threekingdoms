## 1. 한눈에 — 전투 화면의 골격

조조전 전투 화면은 **톱다운 직교 그리드 맵 + 상단 시스템 바 + 우측 패널/플로팅 정보창**으로 구성된다(영걸전 직계). `[스샷]` 실제 화면(「양평관 전투」)으로 확정한 배치:

```
┌────────────────────────────────────────────────────────┐
│ [파일][기능][정보]  ← 윈도우 메뉴                              │
│ [아이콘 툴바 12개]                        ┌──────────┐    │  ← 우상단 미니맵
│ ▌양평관 전투     ▌턴 수 3/30 ▌            │  미니맵   │    │    (적/아군 점)
├──────────────────────────────────────────┴──────────┤    │
│  (맵 = 톱다운 직교 타일)                                  │
│   유닛 스프라이트                       ┌──────────────┐  │
│   ┌─ 유닛정보 팝업(플로팅) ─┐            │ 장수 정보 패널 │  │  ← 우측 상시
│   │ 관우·철기병 Lv99       │            │ 초상+탭:      │  │     (선택 유닛)
│   │ ❤426/632  MP111/111   │            │ 장수열전/부대  │  │
│   │ Exp20   성내 100%(지형) │            │ 특성/능력/장비 │  │
│   └───────────────────────┘            │ /책략         │  │
│   ■ 이동범위=파랑칸  ■ 공격범위=빨강칸     │ 공/정/방/순/사 │  │
│                                        │ 막대 + 이동력  │  │
│                                        │ 앞장수|다음|닫음│  │
│                                        └──────────────┘  │
└────────────────────────────────────────────────────────┘
```

- **상단 시스템 메뉴 바** = `파일/기능/정보` 텍스트 메뉴 + **아이콘 툴바 약 12개**(전군/세이브/설정 등 시스템 명령). `[스샷]`
- **상태 띠** = `스테이지명 | 턴 수 N/최대`(예 "양평관 전투 / 턴 수 3/30"). 좌표·지형도 표시. `[스샷]`
- **우상단 미니맵** = 전장 축소(아군/적 색점). `[스샷]`(원작 SMLMAP/HEXZMAP 추출과 일치)
- **유닛 정보 팝업**(커서 근처 플로팅) = 이름·병종·Lv · ❤HP현/최대 · MP현/최대 · Exp · **지형 보정%(성내 100% 등)**. `[스샷]`
- **장수 정보 패널**(우측) = 초상 + 탭(`장수열전/부대특성/능력/장비/책략`) + 능력 막대(공격력/정신력/방어력/순발력/사기) + 이동력 + `앞의장수/다음장수/닫음`. `[스샷]`
- 상시 자원 바 **없음**. (※ 넥슨 《조조전 Online》만 상단에 군량/은전/금전/공적/명예 5종 자원 바 `[스샷]` 확인 — **모바일 리메이크 전용**, 원작엔 없음. CLAUDE.md §4 "내정·자원생산 금지"와도 충돌 — **베끼지 말 것**.)

`[추출]` **레이아웃 진실 (원본 파일 측정 — 포맷 완전 해독)**:
- 초상(`FACE.E5`) = **64×80px, 8bpp(256색)**, 정확히 **228장** (장수 1인당 1장).
- UI 그래픽(`U_SELECT.E5`) = **8bpp(256색), 폭 400px**, 5청크:
  - chunk 0–2 = **50×50 커서/선택 마커** 스프라이트 (애니 프레임 추정)
  - chunk 3 = **400×296 장식 윈도우 패널** — **청동기 문양(饕餮·蟠龍紋) 부조 + 좌우 전서체(篆書) 카르투슈** + 석재/청동 텍스처 배경 (idx 59–170). **`[스샷]` 이 패널 = 일기토/이벤트 대사 배경으로 실사용 확인**(여포vs유비 일기토 스샷의 배경이 정확히 이 청동 문양) — 추출물과 실화면이 교차검증됨.
  - chunk 4 = **400×128 윈도우 배경 틴트** (idx 61–63, 거의 평면 3톤)
- → **구조적 사실 ①**: 메뉴/정보 창은 "**장식 배경 패널 아트 + 그 위에 동적으로 그리는 텍스트·아이콘·초상**" 합성 구조. U_SELECT는 *글자 박힌 커맨드 바가 아니라 창 크롬(배경)* 이다 → 실제 행동 메뉴 버튼의 정확한 좌표는 여전히 미확정(§9 영상 항목).
- → **구조적 사실 ②(아트 방향)**: UI 크롬이 **청동기 문양/전서체**의 동양 고전 모티프. 우리 수묵 채색 노선과 같은 계열 — 프레임/윈도우 장식의 직접 레퍼런스.

> ⚠️ 정정 이력: 초기엔 U_SELECT를 4bpp(16색)·800폭으로, "UI 16색 vs 초상 256색 레이어 분리"가 설계적 사실이라 적었으나 — 픽셀 디코드 결과 **둘 다 8bpp(256색)** 였다. 그 결론은 폐기.

▷ **우리 적용**: 현재 `apps/web/src/battle/hud/`(ActionMenu·UnitPanel·TurnBanner)는 이미 "가장자리 부유창" 모델 — 방향 일치. 상시 HUD 최소화 원칙만 유지.

## 3. (B) 입력·조작 문법 — 한 유닛의 한 턴

흐름 골격은 `[스샷]`(이동=파랑·공격=빨강 칸, 선택 유닛 정보 패널 동시 표시 확인), 행동 메뉴의 **항목 구성·팝업 좌표만 `[추정]`**(영상 검증 §9).

```
유닛 선택(커서 확정)  →  [스샷] 이동범위=파랑칸 + 우측 장수정보 패널 동시 표시
   │
   ▼
목적 칸 선택  ──[취소]→ 선택 해제
   │
   ▼
행동 메뉴 출현  (항목/좌표 [추정])
   ├─ 공격  → 사정거리 내 적 선택([스샷] 공격범위=빨강칸) → (예상결과 표시?) → 전투 해결
   ├─ 계략  → 책략 리스트(MP 소비) → 대상/범위 조준 → 확정
   ├─ 도구  → 소지품 → 사용  (전투 중 도구함)
   ├─ 특수  → 병종/고유 능력
   └─ 대기  → 턴 종료(그 자리)
```

- **확인/취소 2버튼 커서 모델**: 확인으로 진행, 취소로 한 단계 back. 이동 확정 전이면 **이동 자체도 무를 수 있음**(되돌리기). `[추정]`
- 행동은 유닛당 1턴 1회. 이동 후 공격/계략으로 턴 소모. `[위키]`(턴제 SRPG 표준)
- **간접 공격 무반격** 규약(궁병 2칸·발석차 3칸)은 우리 §7과 동일 — 타겟팅 UX에서 "반격 여부"를 예상창에 표기할지가 포인트. `[추정]`

▷ **우리 적용**: `inputMachine.ts`의 상태(선택→이동→행동→타겟)가 이 흐름과 1:1. 원작과의 차이는 "되돌리기 깊이"와 "예상결과 표시 유무" 정도 — 영상으로 확정할 항목.

## 5. (D) 전투 해결 연출 — 게임의 간판 일기토 포함

### 5-1. 일반 공격 해결 `[위키]`/`[추정]`
- 공격 확정 → (예상 대미지/반격 표시 추정) → 합 연출 → 대미지 수치 팝 → 반격 → 격파 시 부대 소멸 연출.
- 크리티컬·상성 유리 시 추가 연출. 상성: **기병>보병>궁병>기병** (불리측 방어 보정). `[위키]`

### 5-2. 일기토(一騎討ち) — **원작 메커니즘 정밀 확보** `[위키]`
> 출처: tartancheck 일기토 해설 + ja.wikipedia. **이건 우리 §9 일기토 설계의 직접 원본이라 상세히 기록.**

- **발동**: 전투당 **1회**, 인접한 네임드끼리 접촉 시. 숨은 **용맹치(勇猛値)=0** 장수는 **절대 일기토를 걸지 않음** — 사마의·제갈량·육손 등 **고능력 21명**이 용맹 0. (= 책사형은 일기토 안 함)
- **조작 = 天/地/人 가위바위보 카드**. 天>地, 地>人, 人>天. 같은 타입이면 **수치 차이가 대미지**.
  - **必殺(필살)**: 모든 카드값 일시 2배.
  - **反撃(반격)**: 카드값 0이 되지만 들어온 대미지를 **반사** (상대가 상성상 이기는 타입이면 무효).
  - **重ね出し(겹쳐내기)**: 카드 조합/누적 전략.
- **진행**: 최대 **5합**. 각자 사기 **약 100**에서 시작. **남은 사기 높은 쪽 승리**.
- **부대 피해 스케일**: 사기가 2/3로 떨어지면 병력 약 40% 손실, 1/2면 약 50%. 사기가 초기값으로 복귀하는 극히 드문 경우 무승부.
- **보상**: 일기토에서 **이겼을 때만** 일반 격파와 동일 경험치 획득. 지면 0.
- **플레이어 입력 있음**: 매 합 카드 능동 선택.

**`[스샷]` 일기토 화면 구성 확정** (여포 vs 유비 스토리 일기토):
- **배경 = 청동기 饕餮 문양 패널**(=§1에서 추출 해독한 `U_SELECT.E5` 400×296 그 패널). 추출물↔실화면 교차검증.
- **중앙 = 두 장수 합 연출**(기마/도보 스프라이트가 맞붙음).
- **양 모서리 = 화자 초상 + 말풍선**(좌상 여포 "어서, 덤벼라!" / 우하 유비 "하는 수 없군. 얏!"). 스토리 일기토는 대사 스크립트 고정(§9 스토리 일기토 = 영상 컷신 노선의 원형).
- → **우리 적용**: 청동 문양 배경 + 모서리 초상/말풍선 = 우리 스토리 일기토 컷신 연출의 직접 참조. 일반 일기토(天地人 카드)는 별도 화면.

> ⚠️ **우리 설계와의 분기점(기록만, 갭분석 아님)**: CLAUDE.md §9는 *일반 일기토 = 유저 입력 없는 Spine 자동 합연출 / 즉사 없음, 스토리 일기토 = 스크립트 고정 영상*으로 재해석했다. **원작은 일반 일기토도 天地人 미니게임(유저 입력)**이다. 즉 우리는 "원작 일기토의 카드 미니게임을 의도적으로 걷어내고 연출형으로 바꾼" 상태 → 이 결정의 근거/리스크를 §9에서 재확인할 때 본 절을 참조.

### 5-3. 도파민 레이어 원형 `[추정]`
- 격파·일기토 승리·크리티컬이 원작의 "잭팟 순간". CLAUDE.md §12(코인 팝·콤보·잭팟)는 이 순간들을 **연출로 증폭**하는 것 — 원형은 단출하므로 우리가 채우는 영역.

▷ **우리 적용**: 일반/스토리 일기토 2종 분리(§9)는 유지하되, 원작 카드전을 "스토리 일기토 일부에 옵션으로?" 같은 검토는 별도 브레인스토밍.

## 7. (F) 성장·상태가 화면에 묻어나는 방식

- **레벨업**: 행동 기반 경험치. 최대 **Lv50**. `[위키]`
- **병종 랭크업**: **Lv15 → 2단계, Lv30 → 3단계**, **印綬(인수) 아이템** 사용. (영걸전은 아이템 없이 자동인 것과 달리 조조전은 印綬 소비) `[위키]`
- **사기(士気)**: 전투를 유리하게 끄는 핵심 자원. 책략으로 ±. **사기 버프는 행운(luck)을 올리지 않으므로 회복확률엔 영향 없음**. 상태이상 1턴차 탈출 확률 = 행운/2. `[검색]`
- **계략(計略) 분류** `[검색]`: 환성(사기↑)·분기(능력↑: 무관=공격/문관=정신)·고무(능력↑)·견고(방어↑). 도사계=방해계(적 사기·민첩·공·방 ↓), 풍수사계=회복계(아군 회복+버프). 화/수/지/풍 공격계 + **禁呪系**(천후+宝玉 조건, 신화생물 소환). `[위키]`

▷ **우리 적용**: 印綬 소비식 랭크업은 우리 §10 "장비 단계 해금" 및 승급 룰과 비교할 포인트(기록만).
