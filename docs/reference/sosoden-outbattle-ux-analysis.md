## 0. 한눈에 — 막간의 구조

조조전은 **"전투 사이의 모든 것 = 다음 전투 준비"** 라는 철학을 그대로 구현한다(우리 CLAUDE.md §2-4와 동일). 내정·마을탐험 없음. 전투를 클리어하면 결산 후 **출진 준비 흐름**으로 들어간다.

```
전투 클리어
   │
   ▼
[클리어 결산]  경험치·레벨업·획득 아이템/자금 정산
   │
   ▼
[막간 메뉴]  ── 다음 출진 전 준비 허브
   ├─ 군의(軍議)     다음 전투 브리핑 (지형·적·승리조건)
   ├─ 상점           무기/방어구/소모품 구매·매각
   ├─ 편성·장비      능력치 열람 · 장비 착용 · 印綬 승급
   ├─ 출격 멤버 선택  출전 슬롯에 누구를 넣을지
   ├─ 세이브
   └─ 출진 →  다음 전투
```

`[추출]` **레이아웃 진실 (원본 측정)**:
- **게임 기본 해상도 = 640×400** (`LOGO.E5` 풀스크린 청크 256000B = 640×400 8bpp).
- **아이템 아이콘 = 32×32, 8bpp, 104종** (`ITEM.E5` 104청크×1024B). 배경 = **팔레트 인덱스 0(투명)**. 무기(검·도·창·궁·극)/방어구/방패/투구/부채/두루마리 확인.
- 초상(`FACE.E5` 64×80)은 편성·능력치·상점 화면에서도 재사용 추정.

▷ **우리 적용**: 32×32 아이콘 + 투명 인덱스0 규약 = 우리 상점/인벤 아이콘 그리드의 기준 셀 크기. 640×400은 원작 화면비(16:10에 가까움) 참고치 — 우리는 모바일 세로/가로 재설계 대상.

## 1. 클리어 결산 (전투 → 막간 전이) `[추정]`/CLAUDE.md §12

전투 승리 직후의 정산 시퀀스. 경험치 가산 → 레벨업 → 획득 자금/아이템 표시. **우리 §12 "클리어 결산 시퀀스(별 평가→상자 개봉→경험치 바→레벨업)"의 원형**이나, 원작은 단출 — 풀 폴리시는 우리가 채우는 영역.

▷ **우리 적용**: 원작은 "정산 화면" 수준. 우리는 여기에 별 평가(S~C)·도파민 연출을 입힌다(원작에 없는 우리 차별화).

## 2. 군의(軍議) — 출진 전 브리핑 `[위키]`

- 다음 전투 시작 전, **책사가 지형·적·전황을 설명**하는 브리핑 화면. (원작 조조전은 **순욱(荀彧)**이 담당 `[위키]`. → 우리 유비전은 간옹·제갈량 등이 대응)
- 승리/패배 조건, 맵 특성, 주의할 적을 텍스트+초상으로 전달.

▷ **우리 적용**: "출진 전 군의" = 스테이지 목표/기믹을 일러주는 자연스러운 튜토리얼·내러티브 훅. 우리 이벤트 시스템(`eventPlayer.ts`)으로 재현 가능.

## 3. 상점 `[검색]`/`[위키]`

**`[스샷]` 막간 허브 = 단일 "창고(倉庫)" 화면**(창 제목 三國志曹操傳)으로, 하단에 **`출진 / 장비 / 구입 / 매각` 4버튼**이 모드를 전환한다. 즉 상점·인벤·편성이 별도 화면이 아니라 **한 화면의 탭/모드**로 통합돼 있다.

- **창고 일람**(좌측 스크롤 리스트) 컬럼 = `이름 · 속성 · Lv · Exp · 효과 · 장비(○/×)`. 예: 풍신방패(보조·+15%), 맹덕신서, 손자병법서, 적토마, 제갈건, 둔갑천서 등. `[스샷]`
- **우측 = 장수 정보 + 장비 슬롯 3개**(아래 §4). `[스샷]`
- **구입/매각** = 그 4버튼으로. 자금으로 확정 구매·확정 효과(확률 강화·랜덤 스탯 없음 — 원작 그대로). `[스샷]`구조 / `[검색]`내용
- 강화 아이템 맹덕신서(孟徳新書) 등 **레벨업 효율 증폭** 소모품 → 전 장수 만렙 수월. `[검색]`
- 아이콘 32×32(§0). 강한 장비는 장(章) 진행에 따라 점차 입고. `[추출]`/`[검색]`

▷ **우리 적용**: CLAUDE.md §10 "장비 상점제 3슬롯·단계 해금"과 일치. 원작의 **`출진/장비/구입/매각` 단일창 통합**은 모바일 막간 화면(상점→편성→출진)을 한 화면으로 묶는 좋은 레퍼런스.

## 4. 편성·장비·능력치 화면 `[스샷]`

**장수 정보 패널**(창고 화면 우측 + 별도 장수선택 화면 공통) `[스샷]`:
- **기본 능력치**: 무력 · 민첩성 · 지력 · 통솔력 (+ 運/운). ※ 화면엔 4종이 주로 표기되고 운은 내부치.
- **파생/표시 스탯**: HP · MP · Exp · 공격력 · 정신력 · 방어력 · 순발력 · 사기 · 이동력. (전투 문서 §1 장수정보 패널과 동일 필드)
- **장비 3슬롯 확정**: `무기 / 방어구 / 보조`. 각 슬롯에 **개별 Lv·Exp**(예 방천화극 Lv3 Exp0/100 공격력+36 인도공격 / 거울갑옷 Lv3 방어력+34 / 청낭서 매턴 HP회복). → §5 장비 레벨제.
- 네비: `앞의장수 / 다음장수`로 로스터 순회.

**장수 선택(편성·출격) 화면** `[스샷]`:
- **전 장수 그리드**(초상+이름+Lv 격자), 출진 선택 장수는 **빨강 하이라이트**.
- **`출진장수 -N/최대`** 카운터(예 -15/15), **`부대속성순`** 정렬 토글, `결정/취소`.
- 그리드 옆에 위 장수 정보 패널 동시 표시 → 보면서 고름.

▷ **우리 적용**: 기본 4종+운, 파생 스탯, 장비 3슬롯(Lv·Exp)은 우리 `commanders.json`/장비 데이터 설계의 직접 기준. "그리드 선택 + 정보 패널 동시" 편성 UX는 우리 §6 출전 슬롯(8~10명) 화면 레퍼런스. (추출 수치 packages/data/json/sosoden/.)

## 5. 성장이 화면에 드러나는 방식 — **장비 레벨제(핵심 발견)** `[위키]`

> 출처: ja.wikipedia. **우리 §10 "장비 사용 레벨업 3단계"의 직접 원형이라 상세 기록.**

- **장비 개별 레벨제**: 장비품마다 레벨이 있고, **착용자가 전투에서 공격/방어할 때 경험치가 누적** → 레벨업 시 **장비 수치 효과 향상**. `[스샷]` 창고 화면에서 무기/방어구/보조 각 슬롯에 `Lv N · Exp 0/100` 표기 확인.
- **일반 장비 = 최대 Lv3 / 보물(宝物) = 최대 Lv9**. ← **우리 §10 "장비 3단계 단순화"는 원작 일반장비 Lv3 그대로**.
- **만렙 장비는 타입별 아이템으로 매각** 가능.
- **병종 랭크업**: Lv15 이상 + **印綬(인수)** → 클래스2, Lv30 이상 + 印綬 → 클래스3. `[위키]`
- **레벨 상한 = 50** (영걸전은 99 → 조조전에서 50으로 하향). `[위키]`
- **명중률(命中率) 표시**: 영걸전 대비 신요소 — 공격 시 명중률을 보여줌. `[검색]`(전투 화면 요소지만 성장/장비와 직결해 여기 기록)

▷ **우리 적용**: 우리 §10은 "확률 강화·랜덤 스탯 없음 + 장비 3단계 + 레벨캡 스테이지 연동". 원작 장비 Lv제(전투로 경험치 누적)는 **우리의 "노가다 파밍 없음"과 충돌 여지** — 원작은 쓰면 큰다(파밍 유인). 우리는 단계 해금형으로 단순화하므로 *원작의 장비 경험치 누적은 채택 안 함*이 맞다(기록만, 결정은 별도).

## 6. 사실/가상 게이지 (アライメントメーター) `[위키]`/`[검색]`

> 조조전 분기 재미의 핵심 장치. **우리 §5 "사실 모드 본선 + 가상 모드 분기"의 직접 원형.**

- 화면 **우상단**에 **청(青)/홍(赤) 게이지**. 스토리상 발생하는 **선택지**에 따라 비율이 청↔홍으로 기운다. `[위키]`
- **분기 조건**: 어느 한쪽이 **75% 이상** 쏠리면 **가상(仮想) 모드**, 그보다 적으면 **사실(史実) 모드**로 후반 스토리가 갈림. `[검색]`
- 루트: 사실 계열 2개(홍/황) vs 가상 1개(청 — *고대 마왕에 씐 제갈량 저지* 등 사실 이탈 시나리오). `[위키]`
- 게이지 쏠림 자체의 전투 내적 차이는 거의 없고 **보물·추가 전투** 등 약간의 차이. 빨강 우세 시 마지막 전투에서 조건 만족 시 전투 1개 추가. `[검색]`

▷ **우리 적용**: v1은 **사실 모드만**(§15). 게이지 UI·선택지 누적 구조는 **데이터로 미리 깔아두고** 가상 분기는 업데이트로 개방 — 우리 로드맵(§5 "v1은 사실 모드만, 가상 분기는 업데이트")과 정합. 우상단 게이지 = 회차/분기 동력의 시각 장치.

## 9. 후속 (미완)

1. **막간 화면 그래픽 추출** — 상점/편성 화면 배경이 어느 E5에 있는지 미확인. `MCALL*.E5`(이벤트 호출 그래픽, 미해독)·`HM*.E5`(미해독)에 있을 가능성.
2. **전용 팔레트** — ITEM/FACE/UI 모두 구조 확정, 색은 SPALET 차용이라 어긋남(투명=인덱스0 확인). UI/아이템 전용 팔레트 위치 확인 필요.
3. **장비·아이템 수치** — DATA.E5 무기 104종은 추출됨(packages/data/json/sosoden/). 상점 가격·강화 곡선은 EEX/DATA 추가 해독.
