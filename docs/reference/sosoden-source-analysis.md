## 1. 결론 요약

**조조전은 분석 가능하며, 영걸전용으로 만든 추출 툴체인이 그대로 통한다.**

- 영걸전은 `.R3` 컨테이너 + **`Ls11`** 압축, 조조전은 `.E5`/`.EEX`/`.E5S` + **`Ls12`** 압축.
- `Ls12`는 `Ls11`의 다음 버전이고 **압축 알고리즘·컨테이너 레이아웃이 동일**. 기존 추출기(`tools/hero-extract/ls11_extract.py`, LSReader 기반)에서 매직 체크만 `Ls12` 허용으로 바꾸면 코드 변경 없이 작동.
- 90개 `.E5` 중 **21개가 byte-exact로 완전 해제** — 그리고 그 21개가 가장 값진 에셋(초상·유닛 스프라이트·아이템·맵·팔레트·이펙트) 전부.
- 나머지(`HM*.E5`, `MCALL*.E5`, `IMSG.E5`)는 `Ls12` 헤더가 없는 별도 컨테이너로, 추가 리버싱 필요.

검증 스크립트: [`tools/hero-extract/sosoden_probe.py`](../../tools/hero-extract/sosoden_probe.py)

## 3. Ls12 그래픽 포맷 (.E5)

### 헤더 / 레이아웃 (영걸전 Ls11과 동일)

```
0x000  매직 "Ls12" (4C 73 31 32) + 공백(0x20) 패딩 → 0x10까지
0x010  사전(dictionary) 256바이트 (code→literal byte 매핑)
0x110  청크 테이블: 항목당 12바이트 big-endian >III
         comp_size(압축크기), orig_size(원본크기), offset(파일내 데이터 위치)
       comp_size == 0 이면 테이블 종료
```

### 압축 해제 알고리즘 (LSReader / gamma-coded LZSS)

```
get_code():  # Elias-gamma 변형
  code1=0, n=0
  do:  bit=read(1); code1=(code1<<1)|bit; n++  while bit==1
  return code1 + read(n)

decompress(chunk):
  while len(out) < orig_size:
    code = get_code()
    if code < 0x100:            # 리터럴
      out += dic[code]
    else:                       # 백레퍼런스
      move_back = code - 0x100
      copies    = get_code() + 3
      out += out[-move_back] * copies   # 1바이트씩 복사 (겹침 허용)
  # comp==orig 인 청크는 비압축 raw 복사
```

검증 강도: 21개 파일 전 청크에서 출력 길이 == 선언 `orig_size` **그리고** 소비 바이트 == 선언 `comp_size`가 동시에 일치. (FACE.E5는 228청크 연속 일치 — 알고리즘이 맞다는 강한 증거.)

## 3.5 DATA.E5 = 마스터 데이터 테이블 (추출 완료)

`DATA.E5`는 그래픽이 아니라 게임의 **수치 데이터 마스터**. 6개 청크가 각기 다른 레코드 테이블이고,
CP949 한글 이름이 박혀 있어 stride 역산으로 구조가 잡혔다. → `packages/data/json/sosoden/`로 추출.

| 청크 | 테이블 | 레코드 수 | stride | 추출 결과 |
|---|---|---|---|---|
| 0 | **장수** (조조·관우·하후돈·사마의…) | 512 | 32B | `generals.json` |
| 1 | **무기/아이템** (단검·장창·의천검…) | 104 | 25B | `weapons.json` |
| 2 | **범위 셰이프** (책략/무기 AoE 셀 오프셋) | 58 | 36B | `rangeShapes.json` |
| 3 | **병종 클래스** (JOB.TXT 53종 1:1) | 53 | 27B | `classes.json` |
| 4 | **성장 프로파일** ([HP증가30]+[스탯업30]) | 27 | 60B | `growthProfiles.json` |
| 5 | **책략** (초열·업화·탁류·낙석·둔병…) | 73 | 70B | `strategies.json` |

**확정 필드** (원형 삼각측량·구조 검증):
- 장수 능력치 순서 = `mar`무력/`ldr`통솔/`int`지력/`agi`민첩/`luck`운 + `hp` + `mp` (허저 무력49·지력18 / 곽가 무력11·지력49로 식별, 팬에디터 필드순 일치).
- 장수 `classId` = JOB.TXT 번호표 일치. 무기 `power` = 단검5<대검10<강검30 순서.
- 병종 `move`(기병6/보병4/포차3, 승급+1)·`casterType`(참모계=2)·`categoryBit`(스프라이트 원핫).
- **조조전엔 병종별 baseAtk/baseDef 없음** — 공격력은 무력+무기+병종상성 파생(영걸전과 다른 점).

성장: chunk4 = 27 프로파일 × 60B `[hpGrowth 30]+[statGain 30]`, **27개 거의 균일 → 조조전 성장은 병종 차등 작음**(레벨당 ~10 + 소량 스탯업).
책략: chunk5 c11=책략 종류 28종(화·수·지·풍 공격 0~3 + 현혹·유혹·둔병·보급·사신·날씨 등 4+), c13=MP, c15=위력(종류 내 상승), c16=범위셰이프 참조, c35~63=데미지 매트릭스.
범위: chunk2 = 58 셰이프 × 36B, AoE 셀 오프셋 리스트(클수록 길어짐).
미확정: 책략 effectMatrix·셰이프 head·statGain 코드의 정확한 스탯/레벨 매핑(앵커 부족). → `_raw` 동봉.
추출기: `tools/hero-extract/extract_sosoden_data.py` (`decode_classes.py`·`analyze_stats.py`로 분석).

## 4. 기타 포맷

### .EEX — 시나리오/이벤트 스크립트 (117개, 구조+대사 추출 완료)
- 매직 `EEX\0` + 버전 0x0201 + 헤더. **offset 10부터 dword 오프셋 테이블 = 이벤트 블록 인덱스**.
- `R_xx` 59개(이벤트/인트로), `S_xx` 58개(스테이지: [헤더][맵 타일][전투 대사]).
- 텍스트: **CP949 한국어 + Shift-JIS 일본어 명대사** 혼재 (R_00 = "乱世の姦雄、治世の能臣").
- **이벤트 블록 469개 분절 완료** → `events/*.json`. 블록 = `[01 00][ptr][02 00 05 00][제목][00 00 00][명령]`. 제목으로 스토리 구조 노출 (R_16="관우 조조에게 투항한다"·"적토마", R_15="여포의 최후"·"초선 가입").
- **대사 레이어** → `scripts/*.json` (39,725줄). 둘 다 ⚠️ gitignore(코에이 저작).
- **미완**: 명령부 opcode(최빈 0x30·0x2b·0x0d·0x09…) 의미 = VM 스펙 없이 미해독 → 정확한 발동조건(턴/위치/격파/일기토) 디컴파일은 후속. 추출기: `eex_events.py`·`eex_extract.py`.

### .E5S — 유닛/세이브성 데이터 (31개)
- `SV00D/E/S` ~ `SV09D/E/S` 패턴 (10세트 × D/E/S). 헤더 비압축, 뒤이어 CP949 한글 평문 다수 → 일부 즉시 가독.

### TXT (CP949 평문)
- `JOB.TXT`: 병종 53종 번호표 (§6).
- `JOJO.TXT`/`MAIN.TXT`: 팬 에디터 제작 후기/사용법.

### WAV (89개)
- 비압축 PCM. `SE*`(공용), `SE_E_*`(9개), `SE_M_*`(37개) — 즉시 재생/추출 가능.

## 5. 프로젝트 활용

- §4 에셋 파이프라인: 영걸전 외에 조조전 **유닛 스프라이트/초상**도 병종 골격·연출 레퍼런스로 확보 가능.
- §5 시나리오 / §9 일기토: 조조전은 본 프로젝트가 계승하려는 게임 문법(분기·일기토)의 직접 원본. `.EEX` 파싱 시 이벤트/일기토 발동 조건 데이터 참조 가능.
- ⚠️ 레퍼런스/구조 분석 한정. 추출 그래픽·텍스트 원본은 저장소에 커밋하지 않는다(코에이 에셋).

## 7. 재현 방법

```bash
cd tools/hero-extract
python sosoden_probe.py   # 90개 E5 해제 가능 여부 일괄 검증
```

`ls11_extract.py`의 알고리즘과 동일하며, 매직 허용 목록에 `Ls12` 추가 + 대상 경로만 조조전으로 변경한 형태.
