## 1. 분석 대상의 정체 — "맵=영걸전 / 시스템=조조전"의 살아있는 선례

`[영상]` 타이틀바 = `三國志英傑傳`, 한글 메뉴(`파일/기능/정보`), Windows 95/98풍 클래식 창. 640×400급 도트 + 코에이풍 반신 초상화.

**핵심 관찰 — 이 리메이크는 영걸전 무대에 조조전 시스템을 얹었다:**
- **맵/시나리오 = 영걸전**: 사수관·호로관, 유비 삼형제, 화웅·여포, 반동탁연합. 톱다운 직교 시점.
- **전투 시스템 = 조조전**: 병종 클래스명이 조조전 그대로(`의용장/군신/투신/무신/청주병`, 장비 가능 부대 그리드에 `호표기계·무신·군신·기마책사계·선인·맹호대` 등 조조전 JOB 체계 전부 노출, g0077·g0118). 스탯 2계층(기본능력 5 / 파생능력 6), 장비 Lv+Exp 성장, 책략, 일기토 카드배틀 — 전부 조조전 문법.
- → **CLAUDE.md §2-9 우리 v1 전략과 정확히 동일한 조합.** 이 영상은 "그 조합이 실제로 플레이되면 어떤 느낌인가"의 레퍼런스다.

▷ **우리 적용**: 우리 방향이 이미 팬 씬에서 검증·플레이되고 있다는 증거. 단 *독자 브랜딩·아트*는 유지(§CLAUDE.md 법적 라인) — 이건 시스템/UX 흐름 참고용이지 화풍·명칭 모방 대상이 아니다.

## 3. 게임 모드 = 화면 상태 머신 `[영상]`

영상 전체(63분)의 모드 전이를 복원:

```
인트로 페인팅 컷신 → 맵 위 대화 이벤트(낙양궁전/진류막사/회의장)
   → [일기토 튜토리얼: 유비 vs 원술군, 인터랙티브 카드]      ← §12-A
   → 막간 자유이동 모드(NPC 말걸기·상점·세이브 권장)         ← §13
   → 막간 셸(출진/장비/구입/매각)
   → 전투 진입 컷신(승리·패배조건 + 공략 힌트)               ← §14
   → 【사수관의 전투】 턴 루프(아군차례↔적군차례) ……          ← §9~11
        └ [스토리 일기토: 관우 vs 화웅, 스크립트·즉사]        ← §12-B
   → 작전종료 결산(보상금)                                    ← §15
   → 막간 셸(분기 게이지: 호로관 진격/사수관 근교)
   → 【호로관의 전투】 턴 루프 ……
        └ [스토리 일기토: 여포 vs 장비, 스크립트·패배해도 레벨업]
   → 작전종료 결산(보상금 1000)
```

**턴 루프 한 사이클** `[영상]`(g0241~g0264 등 반복):
유닛 호버로 정보 확인 → 유닛 선택(흰 사각) → 이동범위(파랑) 표시 → 이동 → 커맨드 메뉴(공격/대기 등) → 공격 시 대상 호버로 `피해/명중%` 예측 확인 → 실행(인라인 데미지 숫자) → … 전 유닛 행동 후 → `전부대 명령 끝냅니까? 예/아니오` → **풀스크린 "적군차례/아군차례 제N턴" 배너** → 반복.

▷ **우리 적용**: 우리 셸(막간)→전투→결산→막간 루프와 일치. "전투 진입 컷신이 승리/패배조건 + 공략 힌트를 대사로 먼저 깐다"는 패턴은 캐주얼 난이도(CLAUDE.md §11) 구현에 그대로 차용 가능.

## 4. 윈도우 크롬 & 툴바 `[영상]`

- **타이틀바**: 붉은 매 아이콘 + `三國志英傑傳` + Windows 컨트롤(─□✕). 창 포커스/크기에 따라 버튼 노출이 바뀜(활성=3버튼, 비활성=✕만).
- **메뉴바**: `파일 / 기능 / 정보` 3개 고정.
- **툴바 아이콘**: 전투 모드 12~14개 / 막간 모드 좌측 소수만. **회색(비활성)↔컬러(활성)** 토글 — 컨텍스트(유닛 선택 중, 장수 정보 패널 열림 등)에서 점등. 모양 기반 추정 매핑:

| # | 모양 | 추정 기능 |
|---|---|---|
| 1 | 성/관문(노란 지붕) | 거점/본진·메인 (상시 활성) |
| 2 | 건물+아래화살 ↓ (파랑) | 불러오기/입성 |
| 3 | 건물+위화살 ↑ (파랑) | 저장/출진 |
| 4 | 문서/액자 | 부대 일람·명부 |
| 5 | 투구 | 장비/병종 |
| 6 | 주머니(금색) | 도구/아이템 |
| 7 | 산/삼각 | 지형/맵 정보 |
| 8 | 갑옷/장식 | 장비창·능력 |
| 9 | 천막(유르트) | 막사/진영 |
| 10 | `모/공`자 표지 | 작전/진형 (판독 흔들림) |
| 11 | 두루마리/지도 | **전장 지도(미니맵) 토글** |
| 12 | 깃발 | 턴 종료/명령 완료 |

> ⚠️ 개별 기능은 전부 `(추정)` — 클릭 결과를 영상에서 못 봄. 파랑 ↑↓(②③)는 장수 정보 패널이 열렸을 때만 활성 → "앞/뒤 장수 페이징" 가능성도 있음.

▷ **우리 적용**: 윈도우 크롬·클래식 회색 스킨은 *팬 이식의 한계*지 디자인 목표가 아님. 우리는 청동/수묵 프레임으로 대체. 다만 "툴바 컨텍스트 점등" = 가능한 액션만 활성화하는 어포던스는 좋은 패턴.

## 5. 상태 띠 (스테이지 / 턴) `[영상]`

- **좌**: 파란 배지 + 흰 글씨 = **스테이지명** (`사수관의 전투` / `호로관의 전투`). 컷신·막간에선 챕터/출진 라벨(`동탁 타도군 일어나다`, `화웅, 여포 출진`)로 바뀜.
- **우**: 붉은 `턴 수` 라벨 + `현재/최대` (`5/20`). **두 전투 모두 제한 20턴.** 아군 페이즈엔 `턴` 글자칸이 붉게 강조되는 프레임 있음(추정: 페이즈 표시).
- **맨우**: 정적 **풍경 썸네일**(하늘+산+잎). 미니맵 아님 — 분위기 장식.
- **막간 모드**: 상태 띠 자리에 **진행도 분기 게이지 2개**(`호로관 진격` / `사수관 근교`) — 다음 전장 vs 현 위치 진척 표시 (g0206).

## 6. 미니맵 「전장 지도」 `[영상]`

- **별도 자식 창**(타이틀 `전장 지도` + 붉은 매 + 닫기 ✕). 위치 가변(우상단/좌하단), 크기 가변. 드래그·닫기 가능한 부유창.
- **색 규약**: **아군=파랑 점, 적군=빨강 점**(전 구간 다수 일치), **중립/우군=주황 점**(공손찬측). 흰 사각 테두리 = **현재 뷰포트**(메인 화면이 보는 영역, 카메라 팬에 따라 이동).
  - ⚠️ seg07만 "파랑=적?" 의심 — 그러나 다수 프레임·조조전 관례상 **파랑=아군**으로 본다 (추정 정정).

▷ **우리 적용**: [sosoden]·[yeonggeoljeon-data SMLMAP/HEXZMAP 추출]과 일치 — 미니맵은 아군/적 색점 + 뷰포트 박스가 표준. 우리 HUD 미니맵 그대로.

- **능급 등급 + 특성 텍스트에 상성·약점 명시**: "기병계는 보병계에 강하나 간접공격(화살/포차)에 약함", "군신 특성: 책략모방", "무신은 책략에 약함" 등. ▷ CLAUDE.md §7 상성 룰과 정합.

### 8-D. 확인된 주요 병종(조조전 클래스) `[영상]`

## 11. 전투 연출 & 턴 리듬 `[영상]`

- **인라인 전투**: 별도 풀스크린 배틀 컷 **없음**. 맵 위에서 공격 모션 + **피격 유닛 머리 위 흰 외곽선 데미지 숫자**(`21`,`50`,`70`…) + 작은 스파크/별 VFX. 반격 시 작은 빨강 `1` 동반(g0354).
- **데미지는 사전 예측형**: 공격 전 호버 팝업에 `피해 N 명중 N%`를 미리 보여줌 → 실행은 숫자만 팝. (조조전식)
- **격파 연출**: 유닛이 **흰빛+연두 파편으로 흩어지며 소멸**(즉사 아님, 퇴각/격파). ▷ CLAUDE.md §10 "네임드 사망 없음·퇴각만"과 톤 일치.
- **턴 페이즈 배너**: 풀스크린 어두운 틴트 + 대형 흰 글씨 `적군차례` / `아군차례 제N턴`. **진영별 화면 색조 다름**(적=청색조, 아군=흙/세피아조) — 누구 턴인지 색으로 즉시 인지.
- **카메라**: 행동 유닛 따라 부드러운 팬(미니맵 뷰포트로 추적). 명시적 페이드 전환은 거의 없음(컷 위주).

▷ **우리 적용**: 우리는 여기서 *더 나아가야 한다* — CLAUDE.md §2 "표현=2026 풀스택". 인라인 데미지 숫자·턴 배너는 계승하되, 격파 드랍 팝·콤보·크리티컬 잭팟(§12 도파민)을 얹는다. "공격 전 피해/명중 예측"은 캐주얼 난이도 필수 → 반드시 계승.

## 12. 일기토 2종 `[영상]` — 우리 §9 설계와 정합 (단, 이벤트 레이어 = 우선순위 낮음)

> ⚠️ **우선순위 메모**: 일기토는 *연출 이벤트*지 게임성 완성도의 코어가 아니다(길중 판단, 2026-06-14). 우리 §9 설계가 옳다는 걸 확인하는 용도로만 기록하고, 투자는 코어 루프(§7~11) 다음 순번. 과투자 금지.

영상에서 **두 종류의 일기토가 명확히 분리**돼 나타남. 이게 CLAUDE.md §9 "스토리 일기토 vs 일반 일기토" 2종 분리와 그대로 맞아떨어진다.

### 12-A. 일반 일기토 = 인터랙티브 4카드 배틀 `[영상-HD]`
(튜토리얼: 유비 vs 원술 휘하 무장, g0063~0070, t≈625~680s — 풀해상도 확인)

```
┌─ 일기토 전용 배경: 보라/회색 안개 + 동심원 물결 지면 ─┐
│ [원술 초상]                        ♥ 5/5 (HP, 녹색)   │ ← 적: 우상단 게이지
│  (청동 용장식 액자)                ⚑ 1/5 (사기, 청색) │
│  빨강 이름표 "원술"                                    │
│                                                       │
│      [원술 보병]      vs      [유비 기병]              │ ← 중앙 합 애니메이션
│                                          [유비 초상]  │
│   ┌─────────────────────────────────┐    빨강이름표   │
│   │  공격    돌격    방어    반격     │    "유비"      │ ← 4카드(가위바위보식)
│   └─────────────────────────────────┘                │
│ ♥ 5/5  ⚑ 1/5  ← 아군: 좌하단 게이지                   │
└───────────────────────────────────────────────────────┘
```
- **게이지 2종**: HP ♥(최대 5) + 사기 ⚑(최대 5). 적=상단/아군=하단 한 쌍씩.
- **4카드**: `공격 / 돌격 / 방어 / 반격` — 유저가 매 합 선택(상성 + 사기 운용). 공격/돌격으로 사기 소모·HP 깎고, 방어로 사기 충전(추정).
- **연출**: 매 합 양 장수가 중앙 접근해 칼 휘두름. **적=속마음 말풍선**`(흑, 더 신중해야겠다.)`, **아군=외침 말풍선**`하!지금!`. HP 0/5 → KO(쓰러짐 스프라이트).
- 발동 전 **출전 장수 선택 메뉴**(`1.자신 / 2.관우 / 3.장비`, g0060).

→ **이게 우리 "일반 일기토"(시스템, 확률 발동, 유저 개입)**의 선례. 단 영상판은 *카드 수동 선택*까지 받는다(우리 설계는 "유저 입력 없는 Spine 자동 합"이라 더 가벼움).

### 12-B. 스토리 일기토 = 스크립트 컷신 `[영상]`
(관우 vs 화웅 — 즉사, g0196~0202 / 여포 vs 장비 — 패배해도 레벨업, g0372~0378)

**3단 전환**:
1. **VS 인트로 카드**(g0196/g0372): 매 무늬 배경 + 좌상단 적 초상+큰 글씨(파랑) + 중앙 흰 `VS` + 우하단 아군 초상+큰 글씨(빨강).
2. **결투 컷신**(g0197/g0373~376): **흑백 + 동심원 수면/안개 배경**, 양측 청동 액자 초상 고정, 기마 스프라이트 합 애니메이션 + **스크립트 고정 대사**(말풍선 교차). 카드 선택 **없음**(연출만).
3. **결과**(g0200/g0377): 관우vs화웅 → **즉사**(화웅 격파). 여포vs장비 → 장비 패배하지만 `장비의 레벨이 올랐다!`(승패 무관 경험치).

→ 우리 "스토리 일기토"(스크립트 고정, 최고급 컷신, **즉사는 스토리 전용**)와 **정확히 동일한 규칙**. CLAUDE.md §9 "즉사 효과는 스토리 일기토 전용 / 일반 일기토 즉사 없음"이 이 영상에서 그대로 확인됨.

### 12-C. 발동 조건이 공략 = 게임 디자인 `[영상]`
호로관에서 NPC 허자장이 대사로 명시: "여포는 **직접 공격으로 퇴각시키기 거의 불가능**, **12턴에 여포 이동범위 밖으로 빠진 뒤 장비와 일기토를 유도**하라, 일기토를 보면 보상 [쌀]을 주겠다"(g0224~226). → **일기토 발동 조건 자체가 스테이지 공략 요소**(CLAUDE.md §9)임을 실증.

▷ **우리 적용**: 2종 분리·즉사 규칙·발동조건=공략 — **우리 §9 설계가 옳다는 강력한 검증**. 차이점은 표현: 영상판 스토리 일기토는 도트 흑백 컷신이지만, 우리는 Seedance 영상 컷신으로 격상(§3). 일반 일기토도 우리는 Spine 자동 합(영상판의 수동 카드보다 캐주얼). 결투 배경의 "안개+동심원" 무드는 좋은 톤 레퍼런스.

## 13. 막간 셸 — 상점 / 장비 / 세이브 `[영상]`

### 13-A. 막간 메인
하단 4탭 `출진 / 장비 / 구입 / 매각` + 막사 painted 배경 + 아군 장수 스프라이트. 자유이동 모드에선 흰 장갑 손가락 커서로 NPC 호버(네임태그)→말걸기(상점/합류/세이브).

### 13-B. 구입 화면
- **탭**: `무기상`(검) / `도구상`(콩). 
- **상품 테이블**: `이름 | 속성 | 장비 | 효과 | 재고 | 총수 | 가격`(무기) / `이름 | 효능 | 효과 | 재고 | 가격`(도구). `장비` 컬럼 ○/✕ = 현재 선택 장수 장착 가능 여부.
- **우측 장수 미리보기**: 초상 + 병종 + Lv + 무기/방어/보조 현재장비 + 8스탯(HP/MP/공격력/정신력/방어력/순발력/사기/이동력) + `뒤로/다음` 페이징.
- **구입 확정**: 상점주(장비/관우 초상)가 말풍선으로 "N개의 X-금 NNN인데 구입할까요?" + **수치 입력 스피너**(`구입수(1-N)`) + 구입/취소. 소지금 즉시 차감.

### 13-C. 도구 일람
무기일람 탭 = 보유 무기 리스트 + 상세(Lv/Exp/속성/효과/설명) + **`장비 가능 부대` 그리드**(조조전 전 병종 3열, 장착가능=진하게). 통합 리스트 = `이름|소유자|속성|효과|Lv|Exp|특수효과(유/무)`.

### 13-D. 장비 성장
무기/방어구에 **Lv + Exp 게이지**(청룡언월도 Exp 0/200, 가죽갑옷 0/100) — 장비 사용 레벨업. 특수효과: `치명일격공격`(청룡언월도) / `범위공격 이격`(사모) / `인도공격·요극 자동연쇄`(방천화극) / `선제공격`(적토마).
- ⚠️ 영상판은 무기 Exp 성장이 있으나 — 우리 CLAUDE.md §10은 "장비 3단계 단순화·**확률강화/랜덤스탯 없음**". 영상의 Exp 성장은 *확정형*이라 우리 금지선과 충돌 안 함(랜덤 아님). 단순화 여부는 우리 선택.

### 13-E. 세이브 「자료 보존」
`저장할 곳을 선택하십시오` + **슬롯 10개**. 각 슬롯에 **진행 지점 자동 한글 라벨**(`호로관 진격 - 호로관의 전투 - (제4턴)`, Lv 표기). 덮어쓰기 확인 다이얼로그. 일기토/분기 직전 "세이브 권장" 안내(튜토리얼).

▷ **우리 적용**: 막간 4탭 구조·상점 테이블·장비 가능 그리드·수치 입력 스피너는 [sosoden-outbattle-ux-analysis.md]와 교차로 우리 막간 셸의 골격. **세이브 슬롯 자동 라벨(스테이지-턴 기록)**은 우리 클라우드 세이브 UX에 좋은 패턴. 단 우리는 상시 클라우드 세이브라 "세이브 권장 튜토리얼"은 불필요.

## 14. 컷신 / 대화 시스템 `[영상]`

- **2종**: ① **풀스크린 페인팅 컷신**(중요 서술, 하단 풀폭 흰 내레이션 자막, 화자명 없음) ② **맵 위 인게임 이벤트**(대화창 + 초상).
- **대화창**: 둥근 회색 말풍선, **화자명=파랑 글씨**(적/중립도 파랑) + 본문 검정. **초상은 좌/우 코너 가변**(화자 위치 따라 말풍선 꼬리가 초상을 가리킴). 청동/금 액자.
- **타이핑 출력**: 대사가 한 글자씩 점진 출력(g0074 `유`, g0370 `호오`). 속마음은 `( )` 괄호.
- **좌하단 파란 화살표** = "계속/다음" 인디케이터. 유닛 머리 위 작은 말풍선 아이콘 = 발화/대기 표시.
- **전투 진입 브리핑**: 컷신 대사로 **승리조건·패배조건·공략 힌트·병종 상성 튜토리얼**을 깔아줌(허자장·화웅 등). 별도 오버레이로도 표시(`승리조건 / 여포를 퇴각시켜라! / 제한 턴 수 20`, g0227).

▷ **우리 적용**: 화자명 색 + 좌우 코너 초상 + 타이핑 연출은 표준. 우리는 보이스(ElevenLabs)·풀 일러스트로 격상(§3). "전투 전 대사로 조건/힌트 전달" = 캐주얼 온보딩 패턴 계승.

## 15. 결산 (미니멀) `[영상]`

- **작전종료 오버레이**: 맵 위에 큰 회색 외곽 `작전종료` + 금색 `보상금 NNN`(사수관 800, 호로관 1000). **별점·경험치 바·보물 정산 화면 없음** — 영걸전 원작식 미니멀.
- 평가 데이터의 씨앗은 존재: 장수열전 탭의 `출진수 / 퇴각수` 통계.

▷ **우리 적용**: 여기가 **우리가 원작을 가장 크게 넘어서는 지점**. CLAUDE.md §12 "클리어 결산 시퀀스(별 평가→보상 상자 개봉→경험치 바→레벨업) = 풀 폴리시 투자 구간" + §10 "스테이지 평가 S~C(턴/퇴각/보물)". 영상판의 맨숭맨숭한 "보상금 NNN"을 카지노식 결산 시퀀스로 대체하는 게 우리 차별화의 핵심.

## 16. 아트 / 연출 노트 `[영상]`

- **시점**: **톱다운 직교**(쿼터뷰 아님) — 원작 영걸전·조조전과 동일. ▷ CLAUDE.md §3-1 "톱다운 직교" 결정 재확인. (아이소메트릭 폐기가 옳았음을 레퍼런스가 뒷받침.)
- **맵**: painted 배경 위 도트 스프라이트. 성벽·천막·다리·숲·강 전부 배경에 그려넣음(오브젝트 레이어 없음) → CLAUDE.md §3-1 "배경이 풍경 전부 그림, 격자가 이동 막음" 방침과 동일.
- **스프라이트**: SD 비례 도트, 정면/후면 + 기병=기마. 진영색 구분(아군 청/녹/은, 적 청·주황, 네임드 전용색). 격파=흰빛 소멸.
- **초상화**: 코에이풍 회화 반신 + 청동/금 액자. 컷신·장수정보·상점·일기토 공용.
- **회복/드랍 오브젝트**: 맵 위 흰 슬라임/두루마리형 오브젝트(g0091·g0382) → CLAUDE.md "회복 오브젝트"와 일치.
- **폰트**: 비트맵 한글 고딕/명조. 화자명=파랑, 적 이름=빨강, 데미지=흰 외곽, 결산=금색.

## 17. 우리 프로젝트 적용 요약 (▷ 종합)

**그대로 계승 (장르 표준·검증됨):**
- 화면 골격(시스템바 + 부유 정보창 2계층 + 우상단 미니맵), 상시 자원바 없음
- 유닛 정보 2계층(호버 팝업 / 5탭 선택 패널), **스탯 2계층(기본능력 5 / 파생능력 6)**
- 커맨드 8항목(공격/책략/도구/교환/협공/필살/대기/취소) + dim 규칙
- 하이라이트 색(이동=파랑/공격=빨강/커서=흰사각), 공격 전 `피해/명중%` 예측
- 턴 페이즈 배너(진영별 색조), 인라인 데미지 숫자
- **일기토 2종 분리 + 즉사=스토리 전용 + 발동조건=공략** (§9 설계 검증됨)
- 막간 4탭(출진/장비/구입/매각) + 상점 테이블 + 장비 가능 그리드 + 수치 스피너
- 다진영(아군/우군/적군), 지형×병종 보정%

**우리가 넘어서는 지점 (CLAUDE.md §2 "표현=2026 풀스택"):**
- 결산: 미니멀 "보상금 NNN" → **별평가+상자개봉+경험치바+레벨업 풀 시퀀스**(§12)
- 일기토 표현: 도트 흑백 컷신 → **Seedance 영상(스토리) / Spine 자동합(일반)**
- 윈도우 크롬·클래식 회색 UI → **청동/수묵 프레임**(독자 화풍)
- 보이스·BGM·VFX·콤보/잭팟 도파민 레이어

**주의(차용 금지/조정):**
- 코에이/팬 초상·스프라이트·대사 원본 모방 금지(법적 라인). 시스템/흐름만 참고.
- 장비 Exp 성장은 영상에 있으나 우리는 §10 "3단계 단순화" 선택지 — 확률강화/랜덤스탯은 영상에도 없음(우리 금지선과 충돌 없음).
- 일반 일기토 수동 카드(4종)는 우리 설계(자동 합)보다 무거움 — 캐주얼 우선 시 자동화.
