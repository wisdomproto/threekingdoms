## 2. 우선순위 백로그 (코어 게임성 기준 = 정보 제공 + 결정 피드백)

### 🥇 Tier 1 — 전술 의사결정 피드백 (최고 가성비, 엔진 준비됨 → UI 배선 위주)

- **T1-1. 공격 전 피해 예측 표시** — 코어 SRPG의 심장. 캐주얼 난이도(§11 승률 90%+)의 전제.
  - 엔진 **완성**: `computeDamage(ctx, atk, def)` + 반격 `computeDamage(..., counterRatio)` 역방향. 순수 UI 배선.
  - ⚠️ **명중% 넣지 말 것** — 우리 전투는 결정론 100%가 *의도된 설계*(`combat.ts:49` "퍼즐성=계산가능성"). `명중 83%` 대신 **정확한 `피해 N (반격 M)`** 표시 = 리더보드/최소턴 퍼즐과 정합, 레퍼런스보다 *나은* 분기점.
- **T1-2. 호버 정보 팝업(2계층)** — 선택 없이 적/아군 HP·스탯·위협 조회. 현재 선택 패널 1종뿐이라 "적 정보 확인→내 수 계산" 루프가 끊김.
- **T1-3. 적 위협 범위 표시** — 영상의 핵심 공략 장치(여포 위협범위→"12턴엔 빠져라"). 적 호버/선택 시 그 적의 공격 도달 칸을 빨강으로. `getAttackableTargets`로 산출 가능.

### 🥈 Tier 2 — 완성도/정합 (지금 만드는 스테이지 작동에 필요)

- **T2-1. 진영 3종(우군)** — 사수관·호로관에 **공손찬(우군 NPC)** 실제 등장. `Side`에 guest/ally 추가 + 엔진 피아식별·타깃 필터 + 미니맵 주황점. 엔진·데이터·프론트 동시 수정(중간 규모).
- **T2-2. 공격범위 그리드 + 스탯 2계층 패널 노출** — 패널에 기본능력(무력/통솔/지력) + 파생(공격/방어/정신/이동) 병기, 병종 사거리 격자 시각화.

### 🥉 Tier 3 — 폴리시/차별화 백로그 (코어 검증 후)

- 5탭 패널·장수 페이징(정보 위계 폴리시)
- 턴 배너 풀스크린 + 진영 색조 / 커서 흰사각화(연출)
- **교환·협공·필살** — CLAUDE.md §7에서 협공/필살류는 이미 *차별화 백로그*.
