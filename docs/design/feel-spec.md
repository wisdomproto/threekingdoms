## 데이터로 풀 것 (맵/스테이지별로 다른 연출)

**스테이지 JSON에 `camera` 블록 추가** (맵마다 시나리오·스케일이 달라서):
```jsonc
"camera": {
  "zoom": 1.6,            // 초기 줌 (맵 크기/긴장도에 맞춤)
  "focus": [x, y],        // 초기 포커스 칸 (없으면 아군 군주)
  "events": [             // (후속) 이벤트별 카메라 — 일기토/증원 등
    { "on": "duel", "zoom": 2.0, "focus": "actor" }
  ]
}
```
- 맵 에디터/스테이지 작성 시 같이 지정. 기본 휴리스틱: 아군 시작 군집을 적정 줌으로 fit.
- 이벤트 카메라는 .EEX(조조전 이벤트 스크립트) 분석 결과와 우리 이벤트 시스템 결선 시 구현.

## 진행 방식
1. **조조전 플레이/녹화 1패스** → 위 표의 "목표" 칸 구체화(이징 곡선, 줌 배율, UI 위치 등 수치까지).
2. 배치 단위로 burn-down (A 카메라 → B 유닛 → C UI → D 즙). 각 배치 = 한 세션.
3. 새로 발견되는 "세세한 것"은 반응형으로 고치지 말고 **이 표에 추가** 후 배치에 흡수.
