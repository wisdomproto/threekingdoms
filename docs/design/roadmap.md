## 2. 구현 순서 (마일스톤)

### 🎯 M1 — 사수관 수직 완성 (최우선, CLAUDE.md §16)
1스테이지를 **막간→편성→전투→일기토→결산→성장**까지 끊김 없는 풀 루프로. 모든 시스템의 최소 1개씩을 강제해 골격 완성 + 퀄리티 기준점.
- 성장 최소: exp 획득 + 레벨업 (레벨캡 단순).
- 막간 최소: 편성(슬롯 선택) + 상점(장비 1~2종 구매) + 출진.
- 결산: 별평가 → exp바 → 레벨업 연출.
- 스테이지 평가 S~C.
- 보물 1개(사수관 고유) 획득.
- (전투/일기토/배속/자동전투는 이미 done)

### M2 — 시스템 깊이
전투 깊이(도구/장비런타임/책략 확장/상태이상/승급/명중률) · 상점·장비 완성 · 보물 도감 · 기연/떠돌이상인.

### M3 — 콘텐츠 양산
27 스테이지 데이터 · 16 로스터 확정·합류 · 막간 스토리·컷신 · 일기토 10~12. (데이터+에셋 병렬 양산)

### M4 — 메타·폴리시
클라우드세이브/계정 · 회차 · 밸런스 자동화 · 풀 폴리시·도파민 연출.

## 3. 병렬 트랙 (에이전트 할당 단위)

> 같은 파일군을 만지는 트랙끼리만 충돌. 트랙 간 인터페이스(데이터 스키마/이벤트)를 먼저 박으면 병렬도↑.

| 트랙 | 범위 | 병렬성 | 의존 |
|---|---|---|---|
| **A. 엔진/전투** | packages/engine — 도구/책략확장/상태이상/명중률/승급/성장 | 🔶 단일 코어(순차적). 단 기능별 분리 작업 가능 | 스키마 합의 후 B가 데이터 채움 |
| **B. 콘텐츠 데이터** | packages/data/json — 스테이지/로스터/책략할당/보물 JSON | 🟢 **스테이지별·장수별 완전 병렬** (에이전트 다수) | A의 스키마. 맵은 E의 에셋 |
| **C. 막간/메타 UI** | apps/web — 편성/상점/도감/결산/타이틀/스테이지선택 (전투 밖 새 화면) | 🟢 **화면별 병렬**, 전투 코드와 독립 | 장비/자금 스키마(A·B) |
| **D. 연출/스토리** | 막간 스토리 텍스트, intro/outro, 일기토 스크립트, 도파민 연출 | 🟢 스테이지별 병렬 | C(막간 화면)·E(에셋) |
| **E. AI 에셋 생성** | 캐릭터/맵/영상/BGM/SFX/VFX/UI — 생성 파이프라인 | 🟢 **완전 병렬**(유저 생성 + 슬라이스). 코드 무의존 | §4 리스트. 길중이 생성, Claude 슬라이스/결선 |
| **F. 밸런스 자동화** | packages/sim — pacing-gen, 자동 튜닝 루프 | 🟢 독립 인프라 | A(전투 안정), B(스테이지) |

**권장 동시 가동(예)**: 트랙 A가 M1 성장/막간 스키마 박는 동안 → 트랙 B는 로스터 16 데이터 채움 + 트랙 C는 결산/편성 화면 골격 + 트랙 E는 캐릭터 시트 양산. 인터페이스(스키마)만 먼저 합의.

## 4. AI 에셋 생성 리스트 (전수)

> 파이프라인: 길중 생성 → Claude 슬라이스/배경제거/결선. 에셋보드(docs/art/asset-board.html)·맵 청크보드(map-chunk-board.html)에 프롬프트.

## 5. 즉시 착수 (다음 액션 후보)

1. **M1 스키마 합의** — 성장(exp/level), 장비 슬롯, 자금, 보물, 결산 데이터 스키마를 먼저 박는다(트랙 병렬화의 전제).
2. **트랙 B 양산 시작** — 로스터 16 데이터(class/stat/책략/합류) 에이전트 병렬 작성.
3. **트랙 C 골격** — 출진준비(편성→상점→출진) + 결산 화면 골격.
4. **트랙 E 지속** — 캐릭터 시트/맵 청크 생성(길중) → 슬라이스/결선(Claude).
5. **책략 할당표** — 조조전 병종별 책략(웹/자료) 확보 → 트랙 B 데이터.
