# 프로젝트: 삼국지 턴제 전술 SRPG (가칭) — 기획 문서 v1.0

> 이 문서는 프로젝트의 모든 설계 결정의 단일 진실 공급원(SSOT)이다.
> 코드 작업 전 반드시 이 문서를 읽고, 여기 명시된 원칙과 충돌하는 구현은 하지 않는다.
> 변경이 필요하면 문서를 먼저 업데이트한다.

---

## 1. 핵심 컨셉

**"매년 돌아오는 삼국지" — 1998년의 게임성, 2026년의 연출.**

- 코에이 영걸전 시리즈(영걸전/공명전/조조전)의 게임 문법을 계승한 턴제 전술 SRPG
- 캐주얼하게 일주일이면 완주, 회차/난이도/신규 모드로 주기적으로 돌아와 다시 깨는 게임
- 웹 기반 (설치 없음, 링크 하나로 플레이) — 접근성이 핵심 차별화
- 1인 개발 + AI 생성 파이프라인 (일러스트/영상/음성/음악 전부 생성형)

### 포지셔닝
- 조조전 온라인(2016~2020, 넥슨) 서비스 종료 이후 비어 있는 장르의 유일한 현역을 노림
- 수요 근거: 영걸전/조조전 모드 커뮤니티가 20년+ 활성 유지 중. 공급이 끊긴 시장
- 법적 라인: 게임 시스템·역사 인물·실존 전투는 자유. "영걸전" 명칭, 코에이 그래픽/일러스트 스타일 모방, "정신적 후속작" 마케팅 문구는 금지. 독자 브랜딩 필수

### 시리즈 로드맵
1. **1탄: 유비전 1부** (황건적의 난 ~ 적벽, 본 문서의 범위)
2. 1탄 확장: 2부 입촉편, 가상 모드 분기
3. **2탄: 진시황 통일전쟁** — 킹덤(만화) 팬덤이 키운 시장에 정사(史記) 기반으로 진입. 하라 야스히사 창작 요소(킹덤 캐릭터 디자인, 창작 인물)는 절대 사용 금지. 엔진/파이프라인 전면 재사용으로 제작비 1탄의 30% 이하 목표

## 2. 게임 철학 (의사결정 기준)

1. **원작 재현 = 시나리오·맵에 한정. 게임성(전투 시스템)은 결정론 깊이로 최대한 격상.** (2026-06-15 경계 확정 — 종전 "시스템 = 원작 보수주의"를 대체)
   - **시나리오·맵·지형·이벤트·승패조건**: 검증된 원작(영걸전 맵 + 조조전 시스템 문법) 충실 재현. 여기선 재해석 금지.
   - **전투 게임성**: 조조전 문법을 *출발점*으로 삼되, **검증된 뒤 손맛·전술 깊이를 적극적으로 격상**한다 — 필살 게이지(SP 누적→궁극기)·협공/측면 보너스·콤보·병종 패시브 등. 종전 §7 "차별화 백로그(보류)"를 **게임성 격상의 1순위**로 승격.
   - **단, 격상은 전부 결정론(RNG 없음).** 손맛 "대박"은 *확정* 이벤트(상성/협공/약점 적중)에 잭팟 연출을 얹어 만든다. 치명타·명중%·데미지 분산 같은 전투 RNG는 금지 — 밸런스 자동 시뮬(§11)·리더보드/리플레이(§14)·"전투력은 실력으로"(§2-5)가 결정론 위에 서 있기 때문. (§15 금지목록 명문화)
   - **수치 기준점**: 영걸전 추출 수치(docs/reference/yeonggeoljeon-data.md)는 밸런스 기준점 유지. **디자인 목표 = 조조전 온라인** 비주얼(톱다운 + 수묵 채색).
2. **표현 = 2026년 풀스택.** 일기토 영상, 보이스, AI 작곡 BGM, 폴리시 — 원작이 기술 한계로 못 한 것을 채운다
3. **캐주얼 우선.** 기본 난이도는 막힘 없이 완주 가능. 퍼즐적 깊이는 S랭크/챌린지 레이어로 분리. "깨는 건 쉽게, 잘 깨는 건 어렵게"
4. **전투 사이의 모든 것은 다음 전투를 위한 준비여야 한다.** 내정/자원생산/영토관리 절대 금지
5. **랜덤은 재미로, 돈은 확정으로, 전투력은 실력으로.** (BM 절 참조)
6. **보상의 내용물은 설계대로, 전달 방식은 카지노처럼.** (도파민 절 참조)
7. **모든 에셋은 생성 파이프라인의 출력물.** 재생성 가능하게 — 모델이 발전하면 게임이 같이 좋아지는 구조
8. **리텐션 강제 장치 금지.** 스태미나/출석/시즌패스 없음. 떠나는 걸 막는 게임이 아니라 돌아오고 싶은 게임
9. **v1 전략 = 하이브리드 원작 재현 (맵=영걸전 / 시스템=조조전).** (2026-06-13 확정)
   - **맵·지형**: 영걸전 원작 맵을 원천으로 (근거: docs/reference/yeonggeoljeon-data.md §8-2 HEXZMAP 58맵 타일추출, packages/data/json/maps·stages). **지형 기하학=HEXZMAP(정밀)**, **유닛 배치·이벤트·승패조건·증원·보물·일기토 발동조건=영상 분석**(docs/reference/yeonggeoljeon-remake-stages.md, 27스테이지 + 베끼기용 미니맵 downloads/map_refs/). ⚠️ 리메이크는 형주평정으로 1부 종결·적벽은 컷신뿐 → 우리 §5와 후반 구성 다름.
   - **전투 시스템 일체**(병종 체계·능력치·책략·아이템·전투 공식·일기토): 조조전 원작을 **출발점**으로 (근거: docs/reference/sosoden-source-analysis.md, packages/data/json/sosoden/). 조조전이 영걸전 3부작 중 전투 시스템이 가장 정제됨 — 그 시스템을 유비전 무대에 얹는다. ⚠️ **2026-06-15 갱신(§2-1)**: 전투 *게임성*은 여기서 멈추지 않고 결정론 깊이로 격상한다(필살/협공/콤보/패시브). 조조전 재현은 코어 룰의 *베이스라인*이지 천장이 아니다.
   - **UI/UX**: 조조전 원작 화면 문법 분석 = docs/reference/sosoden-battle-ux-analysis.md(전투 화면), sosoden-outbattle-ux-analysis.md(막간·상점·편성). 스크린샷·원본 추출로 레이아웃 확정. **+ 실작동 레퍼런스** = docs/reference/yeonggeoljeon-remake-ux-analysis.md — "맵=영걸전/시스템=조조전" 조합을 그대로 쓴 팬 리메이크 플레이 영상을 프레임 분석(턴 루프·정보 위계·커맨드·일기토 2종·막간 셸의 *동적* UX). 정지 스샷이 못 잡는 게임 진행 흐름의 단일 참조. ⚠️ 넥슨 《조조전 Online》(모바일 리메이크)의 자원바·글로시 HUD는 원작 아님 — §2-1 "디자인 목표=조조전 온라인 비주얼"은 *톱다운+수묵 채색 화풍*만 가리키며 Online의 시스템 UI를 베끼는 게 아니다.
   - **시나리오**: 유비전(§5) 유지.
   - **게임성 격상(결정론 깊이)** = 필살 게이지·협공/측면·콤보·병종 패시브 등은 원작 재현으로 코어 루프가 검증된 뒤 **1순위로 착수**(2026-06-15, §2-1). 병종 재편·모바일 맵 축소는 그 뒤. 아트/연출은 처음부터 독자 노선.

## 3. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 전투 맵 | **톱다운 직교 + 스테이지별 painted 배경 + 보이지 않는 격자 데이터**, PixiJS | 풀 3D 금지. 시점/맵아트 방식은 §3-1 참조 (아이소 타일 방식은 폐기 — 2026-06-13) |
| 캐릭터 애니메이션 | Spine(2D 본 애니메이션) + Spine 런타임 | 파츠 구조로 무기 스킨 실시간 교체 |
| 일기토 연출 | 하이브리드: 일반=Spine 합 연출, 스토리=Seedance 영상 컷신 (+결정타 순간 영상 컷인 테스트) | |
| 프론트 | Next.js / React, 모바일 웹 우선 | 세션 10~15분/스테이지 목표 (v1은 원작 맵 기준 — 측정 후 조정, §11) |
| 백엔드 | Supabase (계정, 클라우드 세이브, 리더보드, 리플레이) | 전투 연산은 100% 클라이언트 |
| 배포/CDN | Vercel + Cloudflare R2(영상, 이그레스 무료) | 영상 트래픽 비용 방지 |
| 일러스트 | Gemini 2.5 Flash Image (nano banana) | 레퍼런스 시트 기반 일관성 |
| 영상 | Seedance I2V | 일기토/이벤트 컷신 |
| 음성 | ElevenLabs | 네임드 일기토 대사 + 내레이션 |
| 음악 | AI 작곡 (Suno/Udio급) | 6~8트랙 + 변형. 상용 라이선스 확인 필요 |
| 효과음 | 무료/저가 SFX 라이브러리 소싱 | AI 생성보다 효율적 |

### 3-1. 시점·맵 아트 방식 (2026-06-13 확정 — 시행착오 끝에 정착)
- **시점: 톱다운 직교** (원작 영걸전·조조전이 그렇다). 아이소메트릭(쿼터뷰)은 시도했다가 폐기 — 원작과 다르고 카메라/정합 비용만 큼.
- **맵 = painted 배경 1장 + 보이지 않는 격자 데이터 1쌍.** 타일을 깔아 예쁘게 만드는 건 천장이 명확(전환·팔레트·반복 한계). 대신:
  1. 격자 지형 데이터(JSON, 진실의 원천)를 **색깔 블록 레이아웃 PNG로 export** (`tools/sprite-pipeline/export_layout.py`, 지형별 색 규약 고정)
  2. 그 블록아웃을 **Gemini(nano banana) img2img**로 "이 배치 그대로 수묵 톱다운 전장으로 칠해" → painted 배경. **그리드가 삽화를 결정**(순서를 뒤집어 위치 어긋남 해결)
  3. 게임은 **격자 데이터로 동작**(이동/상성/충돌), 화면엔 painted 배경. 유닛·하이라이트·VFX는 그 위 스프라이트
  4. `apps/web/public/assets/maps/{stageId}.{webp,png}` 있으면 배경으로 깔고 타일 렌더는 폴백 (webp 우선 — 고해상 배경 용량 절감)
- **2레이어로 확정** (2026-06-13): ① painted 배경(풍경 전부 — 성벽·건물·나무까지 그려넣음) ② 유닛/하이라이트/VFX(그 위 스프라이트) + 비가시 격자(진실의 원천)는 상시 존재. **오브젝트 레이어·깊이정렬 제외** — 유닛은 항상 맨 위, 벽은 격자가 이동을 막고(그림은 배경에 있음), 불·연기 등 동적 연출은 VFX 스프라이트를 배경 위에 얹는다. painted 배경이 풍경을 다 그리므로 별도 오브젝트 스프라이트는 이중작업.
- **스테이지별 카메라 = 데이터**: 맵마다 적정 줌/포커스가 달라 `stage.camera{zoom,focus}` JSON으로 분리(렌더러가 읽어 초기 연출, 미지정 시 기본 1.5+아군 군주). "기본 줌 복귀" 버튼으로 수동 줌/팬 후 복귀. 상세 docs/design/feel-spec.md §A.
- **대형맵**: 청크 분할 + 오버랩 img2img (적벽급 소수만, 대부분 맵은 모바일 다이어트라 통짜 생성 OK).
- **맵 에디터**: 칸 칠하기 → JSON + 블록아웃 export (`tools/map-editor.html`). 27스테이지를 "칠하고→생성" 데이터 작업으로.

### 서버 비용 전망
- 싱글 게임 + 정적 배포 + 비동기 기록 구조라 수천 유저 월 0~5만 원, 수만 유저 월 10~30만 원 수준
- 비용의 본체는 개발 단계 생성 API (1회성, 콘텐츠 비례) — 유저 증가에 비용이 비례하지 않는 건강한 구조

## 4. 에셋 파이프라인

### 캐릭터 생산 체계
1. **레퍼런스 시트** (생성용 원본, 게임 미수록): 전신 3뷰(정면/측면/후면) + 무기 장착 + 표정/포즈 2~3종. 시그니처 무기(관우=청룡언월도 등)는 시트 단계부터 포함 — 장비 반영이 아니라 캐릭터 정체성
2. **쿼터뷰 스프라이트** (시트에서 파생): 2방향(정면비스듬/등비스듬) × 포즈 2~3종(대기/이동/공격) = 4~6장. 좌우는 코드 미러링 (무기 손 바뀜은 장르 관례상 무시)
3. **Spine 리깅**: 파츠 분리 + 무기 슬롯. 병종 단위 골격 규격화 — 같은 병종은 골격 공유, 무기 파츠 호환
4. **일기토 영상 소스** (시트에서 파생)

### 투자 등급
- **풀 제작 (★)**: 유비, 관우, 장비, 조운, 제갈량 + 적측 여포, 조조, 하후돈 — 전용 골격, 일기토 영상, 보이스
- **병종 템플릿**: 나머지 장수는 병종 골격 + 얼굴/색 교체
- 승급 외형 변화는 색/장식 변형 재생성으로 처리

### 스프라이트 골격 총량
v1 = 원작 병종 19종 기준 (§7). 골격 공유(같은 병종 = 같은 골격)로 실제 제작 골격 수는 라인 단위로 압축 — 세부 산정은 원작 재현 검증 후 확정

### 프레임 애니메이션 정책
- 수작업 프레임 애니메이션은 하지 않음. 포즈 전환 + 트윈/이펙트로 연출 (폴리시로 커버)
- 렌더링 레이어는 "유닛 = 애니메이션 시퀀스 재생기"로 설계 — 시퀀스 데이터만 교체하면 업그레이드되는 구조. 향후 생성 AI가 스프라이트 시퀀스를 뽑게 되면 에셋만 교체

## 5. 시나리오 — 1부: 유비전 (27스테이지 + 분기 슬롯)

## 7. 병종 체계 (v1 = 조조전 체계)

§2-9 전환에 따라 v1 병종 시스템은 **조조전 원작**을 원천으로 한다.
원천 데이터: JOB.TXT 병종 번호표 + DATA.E5 추출(docs/reference/sosoden-source-analysis.md §3.5, packages/data/json/sosoden/).

- 조조전 병종 계열(JOB.TXT): 군주·보병·궁병·기병·궁기병·포차·무도가·적병·참모·풍수사·도사·기마책사·무용수 × 3승급 + 특수(서량기병·황건적·해적·곰부대·맹호대·제독·주술사·선인·물자부대 등).
- 상성: 기병계 > 보병계 > 궁병계 > 기병계 (방어력 보정 유리 0.75 / 불리 1.25)
- 간접 공격(궁병계/포차계)은 무반격

- **설계 특성**: 조조전엔 영걸전식 병종별 baseAtk/baseDef가 없다 — 공격력=무력, 방어력=통솔, 정신=지력, 순발=민첩에서 파생(전투 공식은 docs/reference/sosoden-combat-formula.md).
- 영걸전 19종 체계는 docs/reference/yeonggeoljeon-data.md §5에 보존(맵/지형 원천으로는 계속 사용).
- **군주(lord) 병종**: 유비 전용 군주계 추가(unitClasses.json `lord`, line=infantry·move 5·5스탯 등급 A 균형). 유비는 footman→lord 재배정(§6 "유비=군주계" 정합). 상성상 기병에 카운터당해 "보호받아야 하는 전열 표적"으로 긴장 유지. (initialForces.json 원작 추출은 footman 원형 보존 — 재배정은 roster/stage 레이어.)
- **병종↔탈것 시각 매핑**(레퍼런스 영걸전/조조전 문법, 에셋보드 SD 카드 인코딩): 군주=전차 / 기병계=말 / 제갈량(군사)=사륜거 / 보병·궁병·책사·도사 등=도보 / 포차계=공성 병기. SD 스프라이트는 facing=screen-left 고정(코드 미러링 기준).

> 📌 **조조전 분석·이식 진행 상태는 [docs/reference/sosoden-port-status.md](docs/reference/sosoden-port-status.md)에 별도 추적**(추출 산출물·완료/미완·막힌 지점). 진행 노트는 이 CLAUDE.md가 아니라 그 파일에 쓴다 — 병렬 세션 편집 충돌 방지.

### 게임성 격상 (결정론 깊이 — 원작 재현 검증 후 1순위, 2026-06-15)
> §2-1 경계: 시나리오·맵은 재현, 전투 게임성은 RNG 없이 깊이로 올린다. 아래는 우선순위 순.
- ✅ **협공/측면 보너스** (포위 시 확정 추가 피해 + 「협공」 잭팟 연출 + 메뉴 협공 점등). 레퍼런스 §9 근거. *결정론.* **구현 완료.**
- ✅ **병종 패시브 1개씩** (기병 돌격/보병 철벽/궁병 저격 — 자동 발동, 결정론). §8 스킬 2층. **구현 완료.**
- ✅ **연속공격(2중공격)**: 원작 조조전 *순발력 기반 연속공격확률*을 RNG 없이 **이동력 우위**(차 ≥ moveGap)로 치환 — 빠른 병종이 느린 적에게 확정 2회 타격(2타 부분피해). 원작의 일반 시스템 성격 유지(네임드 전용 아님). **구현 완료.**
- ✅ **필살 게이지(SP)**: 레퍼런스 ⚔0/255 누적 게이지의 결정론 구현. 전투 참여(공격·피격·격파)로 SP 누적 → 가득 차면 커맨드 메뉴 「필살」 점등 → *결정론* 대형 일격(무반격, SP 소진) + 「필살!」 배너. **구현 완료.** (네임드별 시그니처 궁극기는 §8 고유 스킬로 후속.)
- ✅ **콤보**: 아군 페이즈 연속 격파 시 확정 보너스 자금(전투력 아님 → 밸런스 중립) + 「콤보 ×N」 연출. 페이즈마다 리셋. §12 도파민. **구현 완료.**
- ✅ **네임드 시그니처 궁극기**(§8 고유 스킬): 필살 발동 시 장수별 이름·위력 — 관우 청룡언월(180%)/장비 장팔사모/조운 단기필마/제갈량 동남풍/유비 인덕의 결의. commanders.json `ultimate{name,percent}` 데이터 구동. **구현 완료.**
- ✅ **아이템 효과 시스템**(§10 보물 "고유 효과 고정"의 실구현): ItemSchema `effects{move,atkPercent,spiritPercent,defensePercent,doubleStrike}`. 말=기동+1(연속공격 임계는 병종 baseMove로 디커플), 보물 14종=고유효과(방어·공격·정신·기동). createBattle 합산→UnitState, computeDamage·doubleStrikes 반영. item-editor가 effects JSON 편집 지원. **구현 완료.** (필살 게이지·콤보는 다음.)
- (그 뒤) 병종 10종+승급 2단계 재편 / 수송대 삭제·인벤토리 공유 / 모바일 맵 축소.

## 8. 스킬 3층 구조

## 10. 성장·아이템

### 레벨
- 행동 기반 경험치 (막타 몰아주기 육성 유지)
- **레벨캡 = 스테이지 진행 연동** (예: 스테이지번호 × 1.5 + 5) — 밸런스 시뮬레이션의 전제조건
- 노가다 파밍 루트 없음

### 장비 (상점제)
- 무기/방어구/보조 3슬롯, 단계 해금은 장 진행 연동
- 장비 사용 레벨업: 3단계로 단순화
- **확률 강화 없음. 랜덤 스탯 없음** (금지 사유: ①밸런스 시뮬레이션 붕괴 ②퍼즐→파밍 게임화 ③리더보드 오염)

### 보물 (스테이지 고유 보상)
- 획득 3문법: ①특정 적 격파/퇴각 ②이벤트 선택 ③숨겨진 조건
- 1부 20~25개, 레벨 없음 고유 효과 고정
- **도감 시스템**: 미획득은 조건 힌트만 표시 — 2회차 동력 + 커뮤니티 공략 떡밥

### 자금
- 수입: 클리어 보상 + 맵 내 상자. 지출: 장비/소모품
- 막간 = 출진 준비 화면 (상점→편성→출진). 마을 탐험 없음

### 사망 룰
- 네임드 사망 없음, **퇴각만** (다음 스테이지 복귀). 퇴각 페널티: 자금 손실 + 평가 하락

## 11. 난이도·회차 설계

- **맵 크기**: v1은 원작 맵 그대로(평균 57×39). 모바일 10~15분 세션용 축소(원작의 50~60%)는 차별화 백로그

### 밸런스 자동화 파이프라인 (개발 핵심 인프라)
> 3개 독립 서브시스템으로 분해(2026-06-15): **A 리포트 카드+게이트 / B 페이싱 생성기 / C 지형 템플릿.**
> A부터 — 측정·게이트 없이 생성기를 만들면 신뢰 불가. 각자 스펙(docs/superpowers/specs/).

- ✅ **A. 밸런스 리포트 카드 + 회귀 게이트 (구현 완료, 2026-06-15)** — `packages/sim/src/reportCard.ts`.
  - **핵심 전환**: 엔진이 무분산 결정론(§2-1)이라 시드 변주는 무의미(결과 1개). 분포는 **{정책 실력 티어 greedy/naive} × {플레이어 레벨 오프셋 −2/0/+2}** 매트릭스로(셀당 1런, 27×6=162런).
  - 스테이지별 분류: HEALTHY/EASY/HARD/**BRITTLE**(g0 승·g+2 패 = 그리디 봇 비단조 아티팩트, 사람은 +2면 더 쉬움 — 17 여남)/IMPASSABLE(진짜 불가). 현재 건강 25·취약 1(17)·쉬움 1(04)·불가 0.
  - **회귀 게이트**(`test/reportCard.test.ts`): 전 스테이지 greedy@0 승 + 불가 제로 + 라벨 스냅샷(`BASELINE_LABELS`). 게임성 격상·데이터 변경이 스테이지를 깨면 **자동 실패**. BRITTLE는 봇 한계라 비차단.
  - CLI `pnpm --filter @tk/sim report-card` → docs/reference/balance-report.md 생성. `runBattle(id, {policy,levelOffset})` 옵션·`naivePolicy`·`withLevelOffset` 추가(하위호환 유지).
- ⏳ **B. 페이싱 커브 기반 적 배치 자동 생성**: 시작→목표 경로 거리를 100%로 정규화, 구간별 전력 비율 배치 (20%지점=전력15%, 50%=40%, 90%=보스). 이벤트 트리거도 경로 %/턴에 바인딩. **+자동 조정 루프**(A 하네스로 목표 수렴 — 봇이 이미 다 이기므로 자동튜닝 용처는 *신규 생성 스테이지*). 미착수.
- ⏳ **C. 지형 문법 템플릿**: "관문 돌파" "협공 방어" "탈출" "화공 이벤트" 등 맵 타입 + 파라미터(크기/병목/증원 턴)로 변형 생성. 완전 랜덤 지형 금지. 미착수.
- 스테이지 정의는 전부 데이터(JSON) — 코드와 분리.

> ⚠️ "승률 90%+"(§11 본문)는 RNG 전제의 잔재 — 결정론 전환(§2-1) 후엔 *분포*가 안 나온다. A의 매트릭스 라벨이 그 자리를 대체(숙련 하한 greedy가 정렙에 깰 수 있는가 = 게이트).

## 12. 도파민 설계 (전달 방식은 카지노처럼)

## 13. BM (광고 기반 F2P + IAP·코스메틱) — 2026-06-14 광고모델 확정

> 1차 수익 = 광고(무료 진입장벽 0 = 웹 접근성 차별화와 정합). 2차 = 광고제거/본편 IAP + 코스메틱.

- **본편**: 챕터 판매 (1~2장 무료 → 이후) 또는 통구매 — 출시 시점 결정. **통구매에 광고제거 포함**.
- **코스메틱**: 캐릭터 스킨 + 무기 스킨. 현금은 **지정 구매만** (확률 상품 금지). 게임 내 재화로는 가챠 가능
- 스킨 노출 설계: 리더보드(스킨 적용 초상화), 리플레이(그 유저 스킨으로 재생 — 기록 자랑+스킨 자랑 동시), 일기토(무기 스킨 메인 무대), 클리어 기록 공유 이미지
- 테마 스킨팩 (수묵화 모드, 도트 레트로 모드 등 — 프롬프트 변형 재생성으로 생산비 ≈ 0)

### 광고 모델 (2026-06-14 확정 — "절제형")
- **리워드 광고(자발적, 도파민 순간에만 — 주력)** 4곳, 전부 opt-in·거부해도 진행 무손실:
  1. **결산 보상 2배** — 별평가→상자→골드 시퀀스 끝에 "광고 보고 골드·기연P 2배" (§12 카지노 결산이 그 자리)
  2. **기연 뽑기 +1회** (§12)
  3. **떠돌이 상인 재입고** (§12)
  4. **상점 골드 충전** — "광고 보고 +골드" (소액·일일 캡). 무기 등 *확정* 장비 구매용
- **전면 광고(로딩) — 절제형(보조)**: 2~3스테이지마다/보스 전만. **막간 스토리 카드(intro/outro)로 번들** = "다음 화 예고", 순수 인터럽트 금지. 빈도 캡 필수.
- **기술**: `AdService` 추상 인터페이스(showRewarded()→Promise<bool> / showInterstitial()) — 우선 stub, 실제 SDK는 **웹게임용**(CrazyGames/GameDistribution/AdinPlay 류) 별도 검증(⚠️ AdSense는 리워드 부적합). 메타스토어 `adFree` 플래그 + 일일 광고 캡.

### 불가침선
- 현금 확률 상품 금지
- 전투력에 영향 주는 모든 랜덤 옵션 금지
- 스태미나/강제 출석/시즌패스 금지
- **광고 동일 원칙**: ①광고로 전투력 랜덤/우위 직접 지급 금지(골드→*확정* 장비만, 밸런스 시뮬 유지) ②광고로 진행 인질 금지(스태미나식 "광고 봐야 계속" 금지) ③강제 전면광고 빈도 캡(절제형) — "떠나는 걸 막는 게 아니라 돌아오고 싶은 게임"(§2-8)
