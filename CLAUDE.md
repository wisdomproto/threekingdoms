# 프로젝트: 삼국지 턴제 전술 SRPG (가칭) — 기획 문서 v1.0

> 이 문서는 프로젝트의 모든 설계 결정의 단일 진실 공급원(SSOT)이다.
> 코드 작업 전 반드시 이 문서를 읽고, 여기 명시된 원칙과 충돌하는 구현은 하지 않는다.
> 변경이 필요하면 문서를 먼저 업데이트한다.

---

## 1. 핵심 컨셉

**"매년 돌아오는 삼국지" — 1998년의 게임성, 2026년의 연출.**

- 코에이 영걸전 시리즈(영걸전/공명전/조조전)의 게임 문법을 계승한 턴제 전술 SRPG
- 캐주얼하게 일주일이면 완주, 회차/난이도/신규 모드로 주기적으로 돌아와 다시 깨는 게임
- 웹 기반 (설치 없음, 링크 하나로 플레이) — 접근성이 핵심 차별화
- 1인 개발 + AI 생성 파이프라인 (일러스트/영상/음성/음악 전부 생성형)

### 포지셔닝
- 조조전 온라인(2016~2020, 넥슨) 서비스 종료 이후 비어 있는 장르의 유일한 현역을 노림
- 수요 근거: 영걸전/조조전 모드 커뮤니티가 20년+ 활성 유지 중. 공급이 끊긴 시장
- 법적 라인: 게임 시스템·역사 인물·실존 전투는 자유. "영걸전" 명칭, 코에이 그래픽/일러스트 스타일 모방, "정신적 후속작" 마케팅 문구는 금지. 독자 브랜딩 필수

### 시리즈 로드맵
1. **1탄: 유비전 1부** (황건적의 난 ~ 적벽, 본 문서의 범위)
2. 1탄 확장: 2부 입촉편, 가상 모드 분기
3. **2탄: 진시황 통일전쟁** — 킹덤(만화) 팬덤이 키운 시장에 정사(史記) 기반으로 진입. 하라 야스히사 창작 요소(킹덤 캐릭터 디자인, 창작 인물)는 절대 사용 금지. 엔진/파이프라인 전면 재사용으로 제작비 1탄의 30% 이하 목표

## 2. 게임 철학 (의사결정 기준)

1. **시스템 = 원작 보수주의.** 전투 룰·보물·성장은 검증된 원작 문법. **시스템 모델은 조조전**(영걸전 룰 + 조조전의 분기·일기토·성장 개선), 영걸전 추출 수치(docs/reference/yeonggeoljeon-data.md)는 밸런스 기준점. **디자인 목표 = 조조전 온라인** 비주얼(톱다운 + 수묵 채색). 재해석 금지
2. **표현 = 2026년 풀스택.** 일기토 영상, 보이스, AI 작곡 BGM, 폴리시 — 원작이 기술 한계로 못 한 것을 채운다
3. **캐주얼 우선.** 기본 난이도는 막힘 없이 완주 가능. 퍼즐적 깊이는 S랭크/챌린지 레이어로 분리. "깨는 건 쉽게, 잘 깨는 건 어렵게"
4. **전투 사이의 모든 것은 다음 전투를 위한 준비여야 한다.** 내정/자원생산/영토관리 절대 금지
5. **랜덤은 재미로, 돈은 확정으로, 전투력은 실력으로.** (BM 절 참조)
6. **보상의 내용물은 설계대로, 전달 방식은 카지노처럼.** (도파민 절 참조)
7. **모든 에셋은 생성 파이프라인의 출력물.** 재생성 가능하게 — 모델이 발전하면 게임이 같이 좋아지는 구조
8. **리텐션 강제 장치 금지.** 스태미나/출석/시즌패스 없음. 떠나는 걸 막는 게임이 아니라 돌아오고 싶은 게임
9. **v1 전략 = 하이브리드 원작 재현 (맵=영걸전 / 시스템=조조전).** (2026-06-13 확정)
   - **맵·지형**: 영걸전 원작 맵을 원천으로 (근거: docs/reference/yeonggeoljeon-data.md, packages/data/json/maps·stages).
   - **전투 시스템 일체**(병종 체계·능력치·책략·아이템·전투 공식·일기토): 조조전 원작을 원천으로 (근거: docs/reference/sosoden-source-analysis.md, packages/data/json/sosoden/). 조조전이 영걸전 3부작 중 전투 시스템이 가장 정제됨 — 그 시스템을 유비전 무대에 얹는다.
   - **시나리오**: 유비전(§5) 유지.
   - 차별화(병종 재편, 모바일 맵 축소, 병종 패시브 등)는 원작 재현으로 재미가 검증된 뒤 별도 단계. 아트/연출은 처음부터 독자 노선.

## 3. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 전투 맵 | **톱다운 직교 + 스테이지별 painted 배경 + 보이지 않는 격자 데이터**, PixiJS | 풀 3D 금지. 시점/맵아트 방식은 §3-1 참조 (아이소 타일 방식은 폐기 — 2026-06-13) |
| 캐릭터 애니메이션 | Spine(2D 본 애니메이션) + Spine 런타임 | 파츠 구조로 무기 스킨 실시간 교체 |
| 일기토 연출 | 하이브리드: 일반=Spine 합 연출, 스토리=Seedance 영상 컷신 (+결정타 순간 영상 컷인 테스트) | |
| 프론트 | Next.js / React, 모바일 웹 우선 | 세션 10~15분/스테이지 목표 (v1은 원작 맵 기준 — 측정 후 조정, §11) |
| 백엔드 | Supabase (계정, 클라우드 세이브, 리더보드, 리플레이) | 전투 연산은 100% 클라이언트 |
| 배포/CDN | Vercel + Cloudflare R2(영상, 이그레스 무료) | 영상 트래픽 비용 방지 |
| 일러스트 | Gemini 2.5 Flash Image (nano banana) | 레퍼런스 시트 기반 일관성 |
| 영상 | Seedance I2V | 일기토/이벤트 컷신 |
| 음성 | ElevenLabs | 네임드 일기토 대사 + 내레이션 |
| 음악 | AI 작곡 (Suno/Udio급) | 6~8트랙 + 변형. 상용 라이선스 확인 필요 |
| 효과음 | 무료/저가 SFX 라이브러리 소싱 | AI 생성보다 효율적 |

### 3-1. 시점·맵 아트 방식 (2026-06-13 확정 — 시행착오 끝에 정착)
- **시점: 톱다운 직교** (원작 영걸전·조조전이 그렇다). 아이소메트릭(쿼터뷰)은 시도했다가 폐기 — 원작과 다르고 카메라/정합 비용만 큼.
- **맵 = painted 배경 1장 + 보이지 않는 격자 데이터 1쌍.** 타일을 깔아 예쁘게 만드는 건 천장이 명확(전환·팔레트·반복 한계). 대신:
  1. 격자 지형 데이터(JSON, 진실의 원천)를 **색깔 블록 레이아웃 PNG로 export** (`tools/sprite-pipeline/export_layout.py`, 지형별 색 규약 고정)
  2. 그 블록아웃을 **Gemini(nano banana) img2img**로 "이 배치 그대로 수묵 톱다운 전장으로 칠해" → painted 배경. **그리드가 삽화를 결정**(순서를 뒤집어 위치 어긋남 해결)
  3. 게임은 **격자 데이터로 동작**(이동/상성/충돌), 화면엔 painted 배경. 유닛·하이라이트·VFX는 그 위 스프라이트
  4. `apps/web/public/assets/maps/{stageId}.png` 있으면 배경으로 깔고 타일 렌더는 폴백
- **2레이어로 확정** (2026-06-13): ① painted 배경(풍경 전부 — 성벽·건물·나무까지 그려넣음) ② 유닛/하이라이트/VFX(그 위 스프라이트) + 비가시 격자(진실의 원천)는 상시 존재. **오브젝트 레이어·깊이정렬 제외** — 유닛은 항상 맨 위, 벽은 격자가 이동을 막고(그림은 배경에 있음), 불·연기 등 동적 연출은 VFX 스프라이트를 배경 위에 얹는다. painted 배경이 풍경을 다 그리므로 별도 오브젝트 스프라이트는 이중작업.
- **스테이지별 카메라 = 데이터**: 맵마다 적정 줌/포커스가 달라 `stage.camera{zoom,focus}` JSON으로 분리(렌더러가 읽어 초기 연출, 미지정 시 기본 1.5+아군 군주). "기본 줌 복귀" 버튼으로 수동 줌/팬 후 복귀. 상세 docs/design/feel-spec.md §A.
- **대형맵**: 청크 분할 + 오버랩 img2img (적벽급 소수만, 대부분 맵은 모바일 다이어트라 통짜 생성 OK).
- **맵 에디터**: 칸 칠하기 → JSON + 블록아웃 export (`tools/map-editor.html`). 27스테이지를 "칠하고→생성" 데이터 작업으로.

### 서버 비용 전망
- 싱글 게임 + 정적 배포 + 비동기 기록 구조라 수천 유저 월 0~5만 원, 수만 유저 월 10~30만 원 수준
- 비용의 본체는 개발 단계 생성 API (1회성, 콘텐츠 비례) — 유저 증가에 비용이 비례하지 않는 건강한 구조

## 4. 에셋 파이프라인

### 캐릭터 생산 체계
1. **레퍼런스 시트** (생성용 원본, 게임 미수록): 전신 3뷰(정면/측면/후면) + 무기 장착 + 표정/포즈 2~3종. 시그니처 무기(관우=청룡언월도 등)는 시트 단계부터 포함 — 장비 반영이 아니라 캐릭터 정체성
2. **쿼터뷰 스프라이트** (시트에서 파생): 2방향(정면비스듬/등비스듬) × 포즈 2~3종(대기/이동/공격) = 4~6장. 좌우는 코드 미러링 (무기 손 바뀜은 장르 관례상 무시)
3. **Spine 리깅**: 파츠 분리 + 무기 슬롯. 병종 단위 골격 규격화 — 같은 병종은 골격 공유, 무기 파츠 호환
4. **일기토 영상 소스** (시트에서 파생)

### 투자 등급
- **풀 제작 (★)**: 유비, 관우, 장비, 조운, 제갈량 + 적측 여포, 조조, 하후돈 — 전용 골격, 일기토 영상, 보이스
- **병종 템플릿**: 나머지 장수는 병종 골격 + 얼굴/색 교체
- 승급 외형 변화는 색/장식 변형 재생성으로 처리

### 스프라이트 골격 총량
v1 = 원작 병종 19종 기준 (§7). 골격 공유(같은 병종 = 같은 골격)로 실제 제작 골격 수는 라인 단위로 압축 — 세부 산정은 원작 재현 검증 후 확정

### 프레임 애니메이션 정책
- 수작업 프레임 애니메이션은 하지 않음. 포즈 전환 + 트윈/이펙트로 연출 (폴리시로 커버)
- 렌더링 레이어는 "유닛 = 애니메이션 시퀀스 재생기"로 설계 — 시퀀스 데이터만 교체하면 업그레이드되는 구조. 향후 생성 AI가 스프라이트 시퀀스를 뽑게 되면 에셋만 교체

## 5. 시나리오 — 1부: 유비전 (27스테이지 + 분기 슬롯)

주인공 시점: **유비** (영걸전 직계). 구조: **사실 모드 본선 + 가상 모드 분기** (조조전의 분기 재미 이식). v1은 사실 모드만, 가상 분기는 업데이트.

### 1장. 황건적의 난 — 튜토리얼 (4)
1. 탁군 의용군 (평지 섬멸 / 이동·공격 튜토리얼)
2. 영천 구원전 (화공 이벤트 튜토리얼, 조조 첫 조우)
3. 광종 전투 (공성 입문)
4. 장각 토벌 (보스전 입문)

### 2장. 반동탁연합 (5)
5. 사수관 (관문 돌파 / 화웅 일기토 — 일기토 데뷔)
6. 호로관 (관문 돌파 대형맵 / 여포전, 삼형제 협공 기믹)
7. 낙양 진입 (시가전 / 화재 지형)
8. 동탁 추격전 (시간제한 / 복병 이벤트)
9. 반하 전투

### 3장. 서주, 여포 (6)
10. 서주 구원전 (방어전)
11. 소패 공방전 (거점 방어)
12. 하비 1차 (패배 이벤트성 탈출전)
13. 원술 토벌 (원문사극 이벤트)
14. 하비 2차 (수공 기믹 / 여포 최후 — 분기점)
15. 허전 사냥 (이벤트 스테이지 — 의대조)

### 4장. 관도~장판파 (7)
16. 관도 탈출
17. 여남 전투 (관우 재회 — 컷신 비중)
18. 박망파 (제갈량 데뷔 화공)
19. 신야 철수전 (백성 호위 이동전)
20. 장판파 (조운 단기필마 — 솔로 스테이지, 시리즈 명물)
21. 장판교 (장비 단신 방어 — 초단기 이벤트)
22. 한진 도하 (관우 구원 합류)

### 5장. 적벽 (5)
23. 강하 방어전
24. 삼강구 (수상전 입문 / 주유 협동)
25. 오림 기습 (황개 고육계 이벤트)
26. 적벽대전 (대형 수상맵 / 화공 대이벤트 — 클라이맥스)
27. 화용도 (최대 분기: 관우의 선택 — 1부 엔딩 분기 + 2부 떡밥)

## 9. 일기토 시스템 (게임의 간판)

1. **스토리 일기토** (스크립트 고정, 원작 방식): 발동 조건/결과 고정, 최고급 영상 컷신. 1부 10~12개 (관우vs화웅, 삼형제vs여포, 장판교 등). 발동 조건 자체가 공략 요소. **즉사 효과는 스토리 일기토 전용**
2. **일반 일기토** (시스템): 네임드 vs 네임드 직접 공격 시 확률 발동 (무력 차/패시브 보정). Spine 자동 합 연출 3~5회, 유저 입력 없음. 승리 시 큰 피해+사기 효과. **즉사 없음**

### 룰

- 연출: 일반=Spine(무기 스킨 실시간 반영) / 스토리=Seedance 영상 / 결정타 순간 1~2초 영상 컷인은 프로토타입 테스트 후 결정

### 영상 물량
스토리 일기토 10~12 × 컷 2~3 = 25~35클립

## 10. 성장·아이템

### 레벨
- 행동 기반 경험치 (막타 몰아주기 육성 유지)
- **레벨캡 = 스테이지 진행 연동** (예: 스테이지번호 × 1.5 + 5) — 밸런스 시뮬레이션의 전제조건
- 노가다 파밍 루트 없음

### 장비 (상점제)
- 무기/방어구/보조 3슬롯, 단계 해금은 장 진행 연동
- 장비 사용 레벨업: 3단계로 단순화
- **확률 강화 없음. 랜덤 스탯 없음** (금지 사유: ①밸런스 시뮬레이션 붕괴 ②퍼즐→파밍 게임화 ③리더보드 오염)

### 보물 (스테이지 고유 보상)
- 획득 3문법: ①특정 적 격파/퇴각 ②이벤트 선택 ③숨겨진 조건
- 1부 20~25개, 레벨 없음 고유 효과 고정
- **도감 시스템**: 미획득은 조건 힌트만 표시 — 2회차 동력 + 커뮤니티 공략 떡밥

### 자금
- 수입: 클리어 보상 + 맵 내 상자. 지출: 장비/소모품
- 막간 = 출진 준비 화면 (상점→편성→출진). 마을 탐험 없음

### 사망 룰
- 네임드 사망 없음, **퇴각만** (다음 스테이지 복귀). 퇴각 페널티: 자금 손실 + 평가 하락
- **스테이지 평가 (S~C)**: 턴 수/퇴각 수/보물 획득 종합 — 긴장감을 영구사망 대신 평가로. 리더보드 직결

## 11. 난이도·회차 설계

- **기본 난이도**: 막힘 없이 일주일 완주. 시뮬레이션 목표 = 승률 90%+, 진행 불가 구간 제로
- **퍼즐 깊이는 분리 레이어**: S랭크 도전 / 챌린지 모드(주간 제약 조건) / 최소턴 기록
- **2회차+ (회차 시스템 = 본체급 기능)**: 레벨/보물 일부 계승 + 적 강화 + 가상 모드 분기는 2회차에 개방
- 난이도 단계 추가 업데이트로 "한 달 뒤 다시 깨기" 지원
- **맵 크기**: v1은 원작 맵 그대로(평균 57×39). 모바일 10~15분 세션용 축소(원작의 50~60%)는 차별화 백로그

### 밸런스 자동화 파이프라인 (개발 핵심 인프라)
1. 페이싱 커브 기반 적 배치 자동 생성: 시작→목표 경로 거리를 100%로 정규화, 구간별 전력 비율 배치 (20%지점=전력15%, 50%=40%, 90%=보스). 이벤트 트리거도 경로 %/턴에 바인딩
2. 지형 문법 템플릿: "관문 돌파" "협공 방어" "탈출" "화공 이벤트" 등 맵 타입 + 파라미터(크기/병목/증원 턴)로 변형 생성. 완전 랜덤 지형 금지
3. AI 자동 플레이 시뮬레이션: 수백 판 자동 실행 → 승률/평균 턴/사망 수 리포트 → 목표치 수렴까지 수치 자동 조정 루프
4. 스테이지 정의는 전부 데이터(JSON) — 코드와 분리

## 12. 도파민 설계 (전달 방식은 카지노처럼)

- **전투 중**: 격파 시 코인/아이템 팝 드랍 연출, 크리티컬·일기토 발동 잭팟 이펙트, 연속 격파 콤보 표시
- **클리어 결산 시퀀스**: 별 평가 → 보상 상자 개봉 → 경험치 바 → 레벨업 연출 (풀 폴리시 투자 구간)
- **기연(奇緣) 시스템**: 클리어 후 기연 포인트로 뽑기 (소모품/자금/소재/희귀 보물/스킨 조각). 짧은 텍스트 연출로 "이야기처럼" 포장. 천장 시스템 포함
- **떠돌이 상인**: 랜덤 등장 + 랜덤 재고 상점
- **스킨 가챠 (게임 내 재화 전용)**: S랭크/도감/챌린지 보상 재화로 뽑기. 등급별 연출 차등, 까기 타이밍 연출, 천장 있음
- **도감 완성 도파민**: 보물/스킨/일기토 영상 갤러리 등록 연출 + 구간 달성 보상

## 16. 첫 마일스톤

**"호로관(또는 사수관) 1스테이지 수직 완성"** — 맵, 유닛, 전투, 일기토, 승리 조건, 결산 연출, BGM까지 전부.
넓은 골격보다 1스테이지의 완성형으로 퀄리티 기준점을 먼저 잡는다.

### 첫 기술 검증 (병행)
1. 관우 레퍼런스 시트 생성 → Spine 리깅 + 무기 파츠 교체 데모 (무기 스킨 BM이 걸린 검증)
2. 같은 시트 → Seedance 일기토 영상 테스트 → Spine vs 영상 하이브리드 비율 결정
3. 전투 엔진 코어 (그리드/이동/상성/턴) + 시뮬레이션 러너 골격

### 작업 분담
- Claude: 엔진, 폴리시, 시뮬레이션, 생성 파이프라인, 스테이지 생성기, 변형 N개 생산
- 길중: 화풍/연출 픽, 재미 최종 판정, 스토리 톤, 변형 중 선택

### 프로젝트 지속성 원칙 (취미 프로젝트 사망 방지)
- 모든 작업은 몇 달 공백 후 복귀 가능하게: 문서화 우선, 데이터-코드 분리, 작은 단위 완결
- 이 문서가 항상 최신 상태를 유지하도록 결정 변경 시 즉시 반영
